The picture control

 

The picture control is one of a number of objects which can make up a window. A picture is generated using the picture command.

 

 

Syntax

 

picture(nr,x,y,w,h)   (eeeee:ddddd)

 

nr ............ control number ID (INT)

x,y ........... top left coordinates of control in pixels (INT)

w,h ........... width and height of control in pixels (INT)

 

Description

 

This command allows the display of a bitmap or jpeg image within a window. The image window can be referred to using a unique ID, nr. Its location in the Prospa window (in pixels) is determined by the (x,y) integer parameters. Its width and height (in pixels) are determined by the (w,h) integer parameters.

 

A variety of picture formats are supported, including JPEG, TIFF, GIF, BMP, and PNG.

 

Additional picture parameters can be set or retrieved using the picture object handle which can be obtained using either of the following methods:

 

obj = picture(ctrlNr,x,y,w,h)

obj = getobj(winNr, ctrlNr)

  

Once the picture object handle has been obtained, you can access the picture parameters using the syntax

 

result = obj->parameter()

obj->parameter(value)

 

where parameter is the name of the parameter you wish to access. The first form gets the current parameter value while the second sets the parameter value.

  

Class function list

 

Following is a list of the available picture parameters. Note that the results of these commands are identical to those obtained using the setpar and getpar functions.

 

All parameters must be prefixed with the object reference operator "obj->" where obj is whatever name you have chosen for the control object. As in the above two examples the "set" version requires a value to be added as an argument while the "get" version will return the value.

 

clearpicture() .................. remove the current picture.

file(STR filename) .............. name of bitmap or jpeg file to load and display

showframe(STR true/false) ....... whether to show a frame for the picture window.

resizetoframe(STR true/false) ... whether the image should be match to fit the picture dimensions.

usebluescreen(STR true/false) ... replace blue pixels with background.

 

Note that resizetoframe and usebluescreen should be defined before the file option, otherwise they will have no effect.

 

In addition, all the generic parameters listed in the control class page can applied to the checkbox object. 

 

Note that changes to the picture will not be made visible until the user calls showwindow for the picture's containing window.

 

Details

 

clearpicture

 

remove the current picture and refresh the control.

 

file

 

file is the file name of the currently displayed image.

 

resizetoframe

 

whether the image should be resized to fit the picture dimensions or just expand or contract to match the picture – default is true.

 

showframe

 

If true, a narrow black border is displayed around the image. If false, no black border is displayed. Defaults to true.

 

usebluescreen

 

If true, then then all blue (0,0,255) pixels will be replaced with the window background color – default is false.

 

Example

 

This example displays a window with a small picture loaded. Note that by default the picture will be scaled to fit the control size but for best image quality and predictable sizing the number of pixels in the two should be matched.

 

 

procedure(Picture test)

 

   n = window("Picture test", -1, -1, 350, 163, "resizable")

 

      # Define all controls with basic parameters

       button(1, 280, 20, 50, 31, "Close", closewindow(0))

       pic = picture(2, 5, 5, 260, 150)

 

   pic->file("test.png")

   showwindow(n)

 

endproc()
