The gridctrl control

  

The gridctrl (grid-control) is one of a number of objects which can make up a window.

 

Syntax

 

gctl = gridctl(objNr, x, y, w, h, cols, rows)      (eeeeeee:ddddddd)

 

nr ............ control number ID

x,y ........... top left coordinates of control in pixels

w,h ........... width and height of control in pixels  

cols, rows..... number of columns and rows in the grid

gctl .......... access to gridcontrol internal functions and parameters (optional)

 

 Description

 

This command adds a grid control to the current GUI window being edited. The grid control can be used for displaying matrices of data, similar to Excel or other spreadsheet applications. The grid control can be referred to using its unique ID, nr. Its location and size in the parent window (in pixels) is determined by the (x,y) (w,h) integer parameters. The number of rows and columns in the grid is set with the cols and rows parameters.

 

Addressing in the grid control is zero-based.

 

Additional grid control parameters can be set or retrieved using the grid control object handle, which can be obtained using either of the following methods:

 

obj = gridctl(ctrlNr,x,y,w,h,cols,rows)

obj = getobj(winNr, ctrlNr)

  

Once the grid control object handle has been obtained, you can access the grid control parameters using the syntax

 

result = obj->parameter()

obj->parameter(value)

 

where parameter is the name of the parameter you wish to access. The first form gets the current parameter value while the second sets the parameter value.

 

Class function list

 

Following is a list of the available gridctl parameters. Note that the results of these commands are identical to those obtained using the setpar and getpar functions.

 

All parameters must be prefixed with the object reference operator "obj->" where obj is whatever name you have chosen for the control object. As in the above two examples the "set" version requires a value be added as an argument while the "get" version will return the value.

 

clear() ..................................... removes all data from this grid control

colheaderheight(INT height) ................. gets or sets the height of the column header row

columns(INT cols) ........................... gets or sets the number of columns in this grid control

colwidth(INT column, INT width) ............. gets or sets the width of a column in this grid control

get(VAR col, VAR row) ....................... gets the value at cell (col,row) in this grid control

label(VAR collabel, VAR rowlabel) ........... gets or sets the header of a row or column

protect(INT col, INT row, "true"/"false") ... gets or sets protection of a cell

rowheaderwidth(INT width) ................... gets or sets the width of the row header column

rows(INT rows) .............................. gets or sets the number of rows in this grid control

set(INT column, INT row, VAR value) ......... sets the value at cell (col, row) in this grid control

showcollabels("true"/"false") ............... gets or sets visibility of column headers

showrowlabels("true"/"false") ............... gets or sets visibility of row headers

 

In addition, all the generic parameters listed in the control class page can applied to the gridctl object.

 

Details

 

clear

 

Remove all data from all cells in this grid control.

 

colheaderheight

 

Get or set the height in pixels of the column headers in this grid control.

 

columns

 

Get or set the number of columns in this grid control. If the number of columns is increased, cells in the newly created columns are blank. If the number of columns is decreased, the cells in those columns are no longer addressable and their data is lost.

 

 colwidth

 

Get or set the width in pixels of a specified column.

 

 get

 

Get the value or values in the specified cell, column, row, or grid.

 

For example, to retrieve the value in the cell at column 5 and row 2 in grid control g, issue

 

     g->get(5,2)

 

The wild card '~' may be used for one or either of the parameters, to retrieve data from entire rows, columns, or the entire grid, as one- or two-dimensional vectors.

 

For example, to retrieve all data from column 7 of grid control g as a one-dimensional vector, issue

 

     g->get(7,~)

 

To retrieve all data from row 1 of grid control g as a one-dimensional vector, issue

 

     g->get(~,1)

 

To retrieve all data from all columns and rows of grid control g as a two-dimensional vector, issue

 

     g->get(~,~)

 

 label 

 

Set the label in the specified column or row header.

 

To set the label of the header of column x of grid control g to value, issue

 

     g->label(x, value)

 

Conversely, to set the label of the header of row x of grid control g to value, issue

 

     g->label(value, x)

 

protect

 

Prevent or enable all interactive and programmatic changes to the specified cell, row, column, or grid. By default, all cells are unprotected.

 

To enable protection of the cell at column 5 and row 2 in grid control g, issue

 

     g->protect(5,2, "true")

 

The wild card '~' may be used for one or either of the parameters, to change the protection state of entire rows, columns, or the entire grid.

 

For example, to disable protection of column 7 of grid control g, issue

 

     g->protect(7,~,"false")

 

To disable protection of row 1 of grid control g, issue

 

     g->protect(~,1,"false")

 

To protect all columns and rows of grid control g, issue

 

     g->protect(~,~,"true")

 

rowheaderwidth

 

Gets or sets the width in pixels of the row headers in this grid control.

 

rows

 

Get or set the number of rows in this grid control. If the number of rows is increased, cells in the newly created rows are blank. If the number of rows is decreased, the cells in those rows are no longer addressable and their data is lost.

 

set

 

Sets the value in the cell at (column, row) to the specified value.

 

showcollabels

 

Gets or sets the visibility of all column headers in this grid control.

 

showrowlabels

 

Gets or sets the visibility of all row headers in this grid control.
