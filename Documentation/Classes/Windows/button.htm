The button control

  

The button control is one of a number of objects which can make up a window. A button is generated using the button command.

 

Syntax 

 

obj = button(nr,x,y,w,h,label,[commandList])   (eeeeeec:dddddss)

 

nr ............ control number ID (INT)

x,y ........... top left coordinates of control in pixels (INT)

w,h ........... width and height of control in pixels (INT)

label ......... label to appear on control (STR)

commandList ... list of commands to execute when button pressed

obj ........... access to button internal functions and parameters (optional)

 

 Description

 

This command adds a button to the current GUI window being edited. The button can be referred to using its unique ID, nr. Its location and size in the parent window (in pixels) is determined by the (x,y) (w,h) integer parameters. The button is identified visually by the string label. Finally, the action taken when the button is pressed is stored in the command list commandList.

 

Additional button parameters can be set or retrieved using the button object handle which can be obtained using either of the following methods:

 

obj = button (ctrlNr,x,y,w,h,label)

obj = getobj(winNr, ctrlNr)

  

Once the button object handle has been obtained, you can access the button parameters using the syntax

 

result = obj->parameter()

obj->parameter(value)

 

where parameter is the name of the parameter you wish to access. The first form gets the current parameter value while the second sets the parameter value.

 

Class function list

 

Following is a list of the available button parameters. Note that the results of these commands are identical to those obtained using the setpar and getpar functions.

 

All parameters must be prefixed with the object reference operator "obj->" where obj is whatever name you have chosen for the control object. As in the above two examples the "set" version requires a value be added as an argument while the "get" version will return the value.

 

icon(STR icon_name) ....... sets the icon to appear instead of a text label.

label(STR text) ........... the text to appear in the button.

mode(STR mode) ............ the button modes.

 

In addition, all the generic parameters listed in the control class page can applied to the button object.

 

Details

 

icon

 

icon_name is the file name of a bitmap (.bmp) in the current directory which stores the icon. The bitmap may be any (reasonable) size and any pixel entries which have the color value RGB[0,0,255] will take the Windows background colour. (Use RGB[0,0,254] if you want blue).

 

mode

 

Buttons may have several special functions specified by the mode parameter.

 

default ...  pressing ENTER on the keyboard is just like pressing this button. The button appears raised to indicate this special mode.

abort ......   halts the current macro and sends an "abort" message to all GUI objects which have a getmessage object defined. This button will remain active in a single threaded environment. It should not be used in a multi-threaded environment; use the generic parameter "active" in this case.

panic ......   halts the current macro and also sends a "panic" message to all GUI object which have a getmessage object defined. This button will remain active in a single threaded environment. It should not be used in a multi-threaded environment; use the generic parameter "active" in this case.

normal ....   button function is normal. This is equivalent to the default mode.

cancel ...   pressing SHIFT-ESC is just like pressing this button. The button function intended to close window as side effect.

  

Example

 

n = window("test",100,100,150,100)

b = button(1,20,20,50,20, "squares",

        pr("\n\n     Squares\n\n");

        for(n = 1 to 20 step 0.5);

            pr("   sqr($n$) = $n^2$\n");

        next(n);

        b->label("pressed");)

showwindow(n)

 

Note the format for commandList. It should be a list of commands separated by semicolons, rather than commas. Line-feeds and spaces between commands comprised by commandList are ignored.
