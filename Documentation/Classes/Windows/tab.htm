The tab control

 

The tab control is one of a number of objects which can be added to a window. A tab control is generated using the tab command.

 

Syntax

 

obj = tab(nr,x,y,w,h,[commandList])   (eeeeec:ddddds)

 

nr ............ control number ID (INT)

x,y ........... top left coordinates of control in pixels (INT)

w,h ........... width and height of control in pixels (INT)

commandList ... list of commands to execute when a tab is selected.

 

Description

 

This command adds a tab control to the current GUI window being edited. The tab control can be referred to using its unique ID, nr. Its location and size in the parent window (in pixels) is determined by the (x,y) (w,h) integer parameters. The action taken when the tab is selected is stored in the command list commandList.

 

The tab control has two parts – the tab itself and its associated page. Controls can be added to the tab page in such a way that they are only visible when the associated tab is selected.

 

Additional tab parameters can be set or retrieved using the tab object handle which can be obtained using either of the following methods:

 

obj = tab(ctrlNr,x,y,w,h)

obj = getobj(winNr, ctrlNr)

  

Once the tab object handle has been obtained, you can access the tab parameters using the syntax

 

result = obj->parameter()

obj->parameter(value)

 

Where "parameter" is the name of the parameter you wish to access. The first form gets the current parameter value while the second sets the parameter value.

 

Class function list

 

Following is a list of the available tab parameters. Note that the results of these commands are identical to those obtained using the setpar and getpar functions.

 

All parameters must be suffixed with the object reference operator "obj->" where obj is whatever name you have chosen for the control object. As in the above two examples the "set" version requires a value be added as an argument while the "get" version will return the value.

 

inittabs(STR tab_list/LIST tab_list) ....... sets the number and names of the tabs

settabs(STR tab_list/LIST tab_list) ........ same as inittabs

renametabs(STR tab_list/LIST tab_list) ..... rename existing tabs

currenttab(STR name/INT number) ............ sets/gets the displayed tab (but not page)

zindex(INT number) ......................... sets/gets the displayed tab by 0-based number

index(INT number) .......................... sets/gets the displayed tab by 1-based number

currentpage(STR name/INT number) ........... sets the displayed tab and tab page

tabname() .................................. gets the name of the displayed tab

tablist(VEC tab_list) ...................... gets the displayed tab and tab page

 

In addition, all the generic parameters listed in the control class page can applied to the tab object.

 

Details

 

inittabs/settabs

 

A list of names is passed which will produced a tab control with the number of tabs equal to the number of names passed e.g.

 

obj->inittabs("Common,T1,T2,Tune") or

 

obj->inittabs(["Common","T1","T2","Tune"])

 

renametabs

 

Accepts a list of names, renaming the respective tab at each index in the list with the name at that index. The list must have the same number of entries as the number of tabs.

 

currenttab/zindex

 

Sets/gets the currently displayed tab but doesn't change the tabpage – this is useful if you want to handle the page drawing yourself (as is done in the Kea macros).  The currenttab may be changed by passing a zero based index or the tab name as a string. In the get mode it returns the zero based index.

 

index

 

Sets/gets the currently displayed tab but doesn't change the tabpage.  The currenttab may be changed by passing a one based index. 

 

currentpage

 

Sets/gets the currently displayed tab and tab page.  The page may be changed by passing a zero based index or the tab name as a string. There is no get mode.  All controls have a tabparent parameter is a row vector which has two parts – a tab control number and a tab control page. By default these values are both -1 and are ignored. However if the tab control number matches an existing control then when you select a tab only those controls which have the corresponding page number will be displayed. This is done automatically when you select the tab manually but can be performed programmatically by setting the currentpage parameter.  The following example generates a tab control with 4 tabs with a control in each tab

 

tabname

 

Gets the name of the current tab.

 

tablist

 

Gets a list of the names of the tabs.

 

Example

 

procedure(test)

 

   n = window("tab test", -1, -1, 230, 121, "resizable")

 

       tb = tab(1, 10, 10, 200, 100)

       bt = button(2, 70, 50, 40, 20, "button")

       cb = colorbox(3, 80, 40, 22, 22, [255,0,0])

       ch = checkbox(4, 130, 60, "no,yes", "no")

       gb = groupbox(5, "groupbox", 50, 40, 120, 56)

 

     # Set other control parameters

       tb->inittabs("A,B,C,D")

       bt->tabparent([1,0])

       ch->tabparent([1,1])

       cb->tabparent([1,2])

       gb->tabparent([1,3])

 

   tb->currentpage(2)

 

   showwindow(n)

 

endproc()
