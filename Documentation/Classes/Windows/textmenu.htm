The textmenu control

 

The textmenu control is one of a number of objects which can make up a window. A textmenu is generated using the textmenu command.

 

Syntax

 

obj = textmenu(nr,x,y,w,h,[commands]) (eeeeec:ddddds)

 

nr ............ control number (INT)

x,y ........... top left coordinates of text in pixels (INT)

w ............. text box width in pixels (INT)

h ............. height of menu in pixels (INT)

command ....... command list to run when menu option selected or return is pressed.

obj ........... access to textmenu internal functions and parameters (optional)

 

 

Description

 

The textmenu is a combination of a text box and a pull-down menu. This command is useful when you need a control which has only certain defined states (such as powers of 2).

 

This command adds a textmenu to the current GUI window being edited. The textmenu can be referred to using its unique ID, nr. Its location and size in the parent window (in pixels) is determined by the (x,y) (w,h) integer parameters. Note that the h entry must be at least large enough to accommodate the complete menu. Calculate h using the following formula:

 

h = 20 + 20*number_of_menu_entries

 

 The textmenu is identified visually by the string label. Finally, the action taken when an entry is chosen from the textmenu is stored in the command list commandList.

 

Additional textmenu parameters can be set or retrieved using the textmenu object handle which can be obtained using either of the following methods:

 

obj = textmenu (ctrlNr,x,y,w,h,command)

obj = getobj(winNr, ctrlNr)

  

Once the textmenu object handle has been obtained, you can access the textmenu parameters using the syntax

 

result = obj->parameter()

obj->parameter(value)

 

where parameter is the name of the parameter you wish to access. The first form gets the current parameter value while the second sets the parameter value.

 

Once the textmenu has been created, you may then add its entries using the class function

 

Class function list

 

Following is a list of the available button parameters. Note that the results of these commands are identical to those obtained using the setpar and getpar functions.

 

All parameters must be prefixed with the object reference operator "obj->" where obj is whatever name you have chosen for the control object. As in the above two examples the "set" version requires a value be added as an argument while the "get" version will return the value.

 

text(STR text) ............... the unevaluated text associated with the currently selected textmenu entry.

value() ...................... the evaluated text associated with the currently selected textmenu entry.

index(INT ix) ................ the 1-based index of the currently selected textmenu entry.

zindex(INT ix) ............... the 0-based index of the currently selected textmenu entry.

menu(VEC entries) ............ comma-separated list of strings; entries in the listbox.

 

In addition, all the generic parameters listed in the control class page can applied to the button object.

 

Details

 

text

 

text is the unevaluated text associated with the current textmenu selection. In its "set" form, selects the first entry with a matching text value.

 

value

 

value is the evaluated text associated with the current textmenu selection.  

 

index

 

index is the 1-based index of the currently selected textmenu entry.

 

zindex

 

zindex is the 0-based index of the currently selected textmenu entry.

 

menu

 

entries is the set of selectable entries in the textmenu, in the form of a vector of strings. See the example below.

 

Example

 

n = window("test",100,100,150,100)

    tm = textmenu(1,20,20,80,100, :selected(););

    tm->menu(["option1","option2","option3","option4"])

    tm->text("option2") # Specify displayed value

    tm->index(2)  # Same as above but uses index in list

showwindow(n)

 

procedure (selected)

  option1 = "First option"

  option2 = "Second option"

  option3 = "Third option"

  option4 = "Fourth option"

  evaluated = tm->value()

  unevaluated = tm->text()

  pr "\nYour choice evaluates to\n\t$evaluated$"

  pr "\nUnevaluated, it is\n\t$unevaluated$"

endproc
