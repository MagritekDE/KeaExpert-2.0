The plot2d control

  

The plot2d control is one of a number of objects which can make up a window. A plot2d is generated using the plot2d command.

 

Syntax

 

plt = plot2d(objNr, x, y, w, h)       (eeeee:ddddd)

 

nr ............ control number ID

x,y ........... top left coordinates of control in pixels

w,h ........... width and height of control in pixels  

plt ........... access to plot internal functions and parameters (optional)

 

 Description

 

This command adds a two-dimensional plot to the current GUI window being edited. The 2D plot "control" can be used for displaying x-y type data. The plot can be divided into multiple regions and each region can display multiple traces. Each trace displays an x-y data set which is stored with the plot control. The plot can be referred to using its unique ID, nr. Its location and size in the parent window (in pixels) is determined by the (x,y) (w,h) integer parameters.

 

Additional plot parameters can be set or retrieved using the plot object handle, which can be obtained using either of the following methods:

 

obj = plot2d(ctrlNr,x,y,w,h)

obj = getobj(winNr, ctrlNr)

  

Once the plot object handle has been obtained, you can access the plot parameters using the syntax

 

result = obj->parameter()

obj->parameter(value)

 

where parameter is the name of the parameter you wish to access. The first form gets the current parameter value while the second sets the parameter value.

 

Class function list

 

Following is a list of the available plot2d parameters. Note that the results of these commands are identical to those obtained using the setpar and getpar functions.

 

All parameters must be prefixed with the object reference operator "obj->" where obj is whatever name you have chosen for the control object. As in the above two examples the "set" version requires a value be added as an argument while the "get" version will return the value.

 

axesmenu(INT menuNr) ............ axes contextual menu.

bkgcolor(VEC colour) ............ plot background colour.

bkgmenu(INT menuNr) ............. background contextual menu.

border("true"/"false") .......... border visibility (true/false)

bordercolor(VEC colour) ......... plot border colour.

draw("true"/"false") ............ enable plot updating (true/false)

keepsubplot(INT x, INT y) ....... keep plot region specified by (one based) x, y indices.

labelmenu(INT menuNr) ........... label contextual menu.

load(STR fileName) .............. load a .pt2 data set into the plot.

multiplot(INT w, INT h) ......... make a new plot with w * h subplots.

OBJ win = parent() .............. get parent window object.

save(STR fileName) .............. save the plot data set to a .pt2 file.

OBJ rg = subplot(INT x, INT y) .. get plot region using (one based) x, y indices.

(INT w, INT h) = size() ......... get the number of columns and rows of subplots.

titlemenu(INT menuNr) ........... title contextual menu.

tracemenu(INT menuNr) ........... trace contextual menu.           

 

In addition, all the generic parameters listed in the control class page can applied to the button object.

 

Details

 

axesmenu

 

Set the plot axes contextual menu.

 

bkgcolor

 

A three-element vector specifying the RGB colour of the plot background.

 

bkgmenu

 

Set the plot background contextual menu.

 

 border

 

Enable and disable border visibility for the plot.

 

 bordercolor

 

A three-element vector specifying the RGB colour of the plot border.

 

 draw 

 

Enable or disable plot drawing. It is sometimes useful to prevent the plot from being updated when it is changed. For example, the plot traces, title, labels, and axes of a plot might all change when a single plot parameter changes. Conceptually and graphically, these operations should all be carried out simultaneously, but by default each of these changes will cause the entire plot to be redrawn. Setting draw("false") before the first "simultaneous" operation, and draw("true") following the final "simultaneous" operation, will ensure that the plot is only drawn once to reflect all of these changes. See draw2d for an example of its use.

 

keepsubplot

 

Reduce a multiplot to a single plot, keeping only the subplot specified by its column and row. For example, in a 3 x 3 multiplot, keep the second subplot in the third row and discard all other plots with plot->keepsubplot(2,3)

 

labelmenu

 

Set the label contextual menu.

 

load 

 

Load and display a previously saved Prospa 2D plt file in the plot window.

 

multiplot 

 

Set the number of subplots in this plot with two integers specifying the number of columns and rows of subplots. For example, to create a plot with three stacked subplots, request plot->multiplot(1,3). Note that multiplot will clear all existing subplots in the affected plot.

 

parent

 

Get the window that contains the plot.

 

save 

 

Save the current plot and all subplots to a specified plt file.

 

subplot  

 

Get a reference to a subplot by its 1-based column and row indices. For example, to select the first subplot in the third column, request plot->subplot(3,1).

 

size 

 

Get a two-element vector indicating the number of columns and rows of subplots in the addressed plot. For example, a plot with three columns and two rows of subplots will have a size of [3,2], for a total of six subplots.

 

titlemenu 

 

Set the title contextual menu.

 

tracemenu 

 

Set the trace contextual menu.
