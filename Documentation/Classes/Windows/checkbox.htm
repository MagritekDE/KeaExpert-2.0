The checkbox control

 

The checkbox control is one of a number of objects which can make up a window. A checkbox is generated using the checkbox command.

 

Syntax

 

obj = checkbox(nr,x,y,states,init,[command]) (eeeeec:dddsss)

 

nr ............ control number ID (INT)

x,y ........... top left coordinates of control in pixels (INT)

states ........ the two states separated by a comma (STR)

init .......... the initial state – must be one of 'states' (STR)

commandList ... list of commands to execute when checkbox is pressed

obj ........... access to checkbox internal functions and parameters (optional)

 

 

Description

 

This command adds a checkbox to the current GUI window being edited. The checkbox can be referred to using a unique ID, nr. Its location in the window (in pixels) is determined by the (x,y) integer parameters. The checkbox can have two states, indicated by the comma delimited string list states (off state, then on state). The string init identifies the initial state. Finally an optional action commandList may be taken when the checkbox state is toggled.

 

Additional checkbox parameters can be set or retrieved using the checkbox object handle which can be obtained using either of the following methods:

 

obj = checkbox (ctrlNr,x,y,states,init)

obj = getobj(winNr, ctrlNr)

  

Once the checkbox object handle has been obtained, you can access the checkbox parameters using the syntax

 

result = obj->parameter()

obj->parameter(value)

 

where parameter is the name of the parameter you wish to access. The first form gets the current parameter value while the second sets the parameter value.

 

Class function list

 

Following is a list of the available checkbox parameters. Note that the results of these commands are identical to those obtained using the setpar and getpar functions.

 

All parameters must be prefixed with the object reference operator "obj->" where obj is whatever name you have chosen for the control object. As in the above two examples the "set" version requires a value to be added as an argument while the "get" version will return the value.

 

text(STR icon_name) ....... the text associated with the current checkbox state.

value(STR text) ........... the integer associated with the current checkbox state (0,1).

init(STR mode) ............ the text associated with the initial checkbox state.

options(STR mode) ......... two-element vector of the text associated with the "unchecked" and "checked" checkbox modes.

 

In addition, all the generic parameters listed in the control class page can applied to the checkbox object.

 

Details

 

text

 

text is the text associated with the current checkbox state.

 

value

 

value is the integer associated with the current checkbox state. If the checkbox is currently unchecked, value is 0. If the checkbox is currently checked, value is 1.

 

init

 

init is the text associated with the initial checkbox state. It is always equal to the value set as the init parameter when the checkbox was instantiated.

 

options

 

options is a two-element vector whose first element is the text associated with the "unchecked" state of the checkbox. Its second element is the text associated with the "checked" state of the checkbox.

 

Example

 

n = window("test",100,100,150,100)

c = checkbox(1,20,20, "no,yes", "no",

           text = c->text();

           val  = c->value();

           in   = c->init();

           opt  = c->options();

           pr ("\nCheckbox text is $text$\n");

           pr ("\nCheckbox value is $val$\n");

           pr ("\nCheckbox init is $in$\n");

           pr ("\nCheckbox options is $opt$\n");

 

)

showwindow(n) 

 

Note the form of commandList. It should be a list of commands separated by semicolons, rather than commas. Line-feeds and spaces between commands comprised by commandList are ignored.
