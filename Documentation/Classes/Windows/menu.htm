The menu control

 

Generate a menu for a user defined window.

 

Syntax

 

menu(object_number, label[, item[, command})  (eeec:dsss)

 

object_number ... unique number for this object (1 based) (INT)

label ........... name of menu – will appear on menu bar (STR)

item ............ menu item label plus keyboard accelerator. (STR)

command ......... command to execute when item is selected.

 

Description

 

User defined GUI windows can have menus. The menus are first formed using this command, and then added to the parent window using setwindowpar or the object addressing notation. The menu consists of:

 

·         a label which appears in the menu bar;

·         pairs of strings which represent the menu item labels;

·         commands which will be called when the item's label is selected.

 

The item string can also include a keyboard accelerator. This is done by ending the label with a tab character \t and then immediately appending the accelerator code. Possible codes are:

 

character ......... ASCII character or those shown below

Ctrl+character .... control key with character

Shift+character ... shift key with character

Alt+character ..... alt key with character

 

Possible characters are the normal ASCII characters plus:

 

Function keys . F1 ... F14

Escape key .... ESC

Tab key ....... TAB

Enter key ..... RETURN

Left arrow .... LEFT

Right arrow ... RIGHT

Up Arrow ...... UP

Down arrow .... DOWN

Home key ...... HOME

Insert key .... INSERT

Delete key .... DELETE

Pause key ..... PAUSE

 

Additional menu parameters can be set or retrieved using the menu object handle which can be obtained using either of the following methods:

 

obj = menu (ctrlNr, label)

obj = getobj(winNr, ctrlNr)

  

Once the menu object handle has been obtained, you can access the menu parameters using the syntax

 

result = obj->parameter()

obj->parameter(value)

 

where parameter is the name of the parameter you wish to access. The first form gets the current parameter value while the second sets the parameter value.

 

Class function list

 

Following is a list of the available menu parameters. Note that the results of these commands are identical to those obtained using the setpar and getpar functions.

 

All parameters must be prefixed with the object reference operator "obj->" where obj is whatever name you have chosen for the control object. As in the above two examples the "set" version requires a value be added as an argument while the "get" version will return the value.

 

VEC menu() ............... get a vector of strings of the menu labels

INT nritems() ............ get the number of items in the menu

STR menuname(STR name) ... get or set the name of the menu

checkitem(INT item) ...... draw a checkmark next to the specified menu item

uncheckitem(INT item) .... remove any checkmark next to the specified menu item

 

In addition, all the generic parameters listed in the control class page can applied to the menu object.

 

Details

 

menu

 

Returns a vector containing all menu labels.

 

nritems

 

Returns the number of labels in the menu.

 

menuname

 

Gets or sets the name of the menu.

 

checkitem

 

Draw a checkmark next to the menu item specified by the parameter, a 0-based index.

 

uncheckitem

 

Remove any checkmark next to the menu item specified by the parameter, a 0-based index.

 

Example

 

 

 n = window("menu test", 100, 100, 200, 100)

 

   button(1,10,10,50,30,"Close",closewindow(0);)

 

    menu(2,"Edit","Copy\tCtrl+C",

                   pr ("Copy selected\n"),

                  "Paste\tCtrl+V",

                   pr ("Paste selected\n"),

                  "Cut\tCtrl+X",

                  pr ("Cut selected\n");)

    menu(3,"Window","Hide\tF2",hidewindow(0);)

 

    setwindowpar(n,"menu",[2,3])

    setwindowpar(n,"show_menu","false")

 

    showwindow(n)

 

 

 

In this case, there are two menus defined. The Edit menu provides the usual copy, paste, cut options while the Window menu allows the F2 key to hide the window. Note that these menus override the normal accelerators which work with GUI windows, which is why the usual pop-up menu has been disabled and the F2 hide command has been added.
