The updown control


The updown control is one of a number of objects which can make up a window. An updown is generated using the updown command.

 

Syntax

 

obj = updown(nr,x,y,w,h,orientation[,commandList])      (eeeeeec:dvvvvss)

 

nr ............ control number ID (INT)

x,y ........... top left coordinates of control in pixels (INT)

w,h ........... width and height of control in pixels (INT)

orientation ... "vertical" or "horizontal" layout of control (STR)

commandList ... list of commands to execute when buttons are pressed

obj ........... access to button internal functions and parameters (optional)

 

Description

 

This command adds an updown control to the current GUI window being edited. This control can increment or decrement a number between two limits with a specified step size. The control can be referred to using its unique ID, nr. Its location and size in the parent window (in pixels) is determined by the (x,y) (w,h) integer parameters. The control orientation can be either horizontal or vertical. Finally the action taken when the control is activated is stored in the command list commandList.

 

Additional updown parameters can be set or retrieved using the updown object handle which can be obtained using either of the following methods:

 

obj = updown (ctrlNr,x,y,w,h,label)

obj = getobj(winNr, ctrlNr)

  

Once the updown object handle has been obtained, you can access then  parameters using the syntax

 

result = obj->parameter()

obj->parameter(value)

 

where parameter is the name of the parameter you wish to access. The first form gets the current parameter value while the second sets the parameter value.

 

Class function list

 

Following is a list of the available updown parameters. Note that the results of these commands are identical to those obtained using the setpar and getpar functions.

 

All parameters must be prefixed with the object reference operator "obj->" where obj is whatever name you have chosen for the control object. As in the above two examples the "set" version requires a value be added as an argument while the "get" version will return the value.

 

base .......... Lowest value the control can take (FLOAT)

nrsteps ....... Number of steps the control can make before reaching its limit. (INT)

orientation ... Either "vertical" or "horizontal" (STR)

stepsize ...... How large each step will be. (FLOAT)

value ......... The current control value (FLOAT)

  

Example

 

This example displays a window with an updown control, a close button, and some static text showing the current value of the updown control.

 

procedure(updown_test)

 

  n = window("updown test", -1, -1, 220, 58)

 

      # Define all controls with basic parameters

       button(1, 140, 15, 50, 30, "Close",closewindow(0);)

       c = updown(2, 80, 16, 16, 30, "vertical",

            c = getobj(0,2);

            pr c->value();)

       statictext(3, 69, 24, "right", "Select value")

       statictext(4, 119, 24, "right", "5")

   showwindow(n)

 

# Set other control parameters

c = getobj(n,2)

c->base(-10)

c->value(5)

c->stepsize(1)

c->nrsteps(21)

 

endproc()

 

Note the format for the command list. It should be a list of commands separated by semicolons, rather than commas. Line feeds and spaces between commands are ignored.
