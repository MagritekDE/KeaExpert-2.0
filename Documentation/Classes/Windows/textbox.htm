The textbox control

 

The textbox control is one of a number of objects which make up a window. A textbox is generated using the textbox command.

 

Syntax

 

obj = textbox(nr,x,y,w,[commands])    (eeeec:dddds)

 

nr ............ control number (INT)

x,y ........... top left coordinates of text in pixels (INT)

w ............. text box width in pixels (INT)

command ....... command list to run when return pressed.

obj ........... access to button internal functions and parameters (optional)

 

Description

 

This command adds a textbox to the current GUI window being edited. The textbox can be referred to using its unique ID, nr. Its location and size in the parent window (in pixels) is determined by the (x,y) (w,h) integer parameters. The action taken when the value in the textbox changes is stored in the command list commandList.

 

Additional textbox parameters can be set or retrieved using the textbox object handle which can be obtained using either of the following methods:

 

obj = textbox (ctrlNr,x,y,w,h,command)

obj = getobj(winNr, ctrlNr)

  

Once the textbox object handle has been obtained, you can access the textbox parameters using the syntax

 

result = obj->parameter()

obj->parameter(value)

 

where parameter is the name of the parameter you wish to access. The first form gets the current parameter value while the second sets the parameter value.

 

Class function list

 

Following is a list of the available textbox parameters. Note that the results of these commands are identical to those obtained using the setpar and getpar functions.

 

All parameters must be prefixed with the object reference operator "obj->" where obj is whatever name you have chosen for the control object. As in the above two examples the "set" version requires a value be added as an argument while the "get" version will return the value.

 

text(STR contents) .... the unevaluated contents of the textbox

password(STR onoff) ... whether the textbox's contents are to be obscured as they are entered ("on"/"off")

debug(STR choice) ......... whether the textbox is part of a debugger ("true"/"false").

value() ............... the evaluated contents of the textbox.

 

In addition, all the generic parameters listed in the control class page can applied to the button object.

 

Details

 

text

 

text is the unevaluated text associated with the current textbox contents. 

 

password

 

When on, each character entered in the textbox is displayed as an asterisk character.

 

debug

 

choice may be "true" or "false". If true, the editor is used as part of a debugger.

 

value

 

value is the evaluated text associated with the current textbox contents.

  

Example

 

n = window("Login",100,100,150,80)

   statictext(1,10,13,"left","USER")

   user = textbox(2,50,10,80)

   statictext(3,10,38,"left","PASS")

   pass = textbox(4,50,35,80,:checklogin(user,pass);)

   pass->password("on")

   result = statictext (5,10,60,"left","")

showwindow(n)

 

procedure(checklogin, user, pass)

   uname = "magritek"

   unamepass = "ketirgam"

   if (uname == user->text())

      if (unamepass == pass->text())

         result->text("Logged in.")

         return

      endif

   endif

   result->text( "Not logged in.")

endproc()
