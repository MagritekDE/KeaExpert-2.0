The slider control

    

The slider control is one of a number of objects which can make up a window. A slider is generated using the slider command.

 

Syntax

 

slider(nr,x,y,w,h,orient,command) (eeeeeec:dddddss)

 

nr ............ control number (INT)

x,y ........... top left coordinates of slider in pixels (INT)

w,h ........... width and height of slider in pixels

orient ........ slider direction: "horiz" or "vert" (STR)

command ....... commands to run when slider adjusted

 

Description

 

This command adds a slider to the current GUI window being edited. A slider is like a scroll-bar, except that it invokes the callback procedure it is controlling when the slider control is moved. It mimics the slider controls found on stereos, etc. The slider can be referred to using its unique ID, nr. Its location and size in the parent window (in pixels) is determined by the (x,y) (w,h) integer parameters. orient is the orientation of the slider (horiz or vert) and command is a semicolon delimited command list which will be executed each time the slider is moved. Note that the slider range defaults to [0, 100] unless you use the range function of the slider object.

 

Additional slider parameters can be set or retrieved using the slider object handle which can be obtained using either of the following methods:

 

obj = slider (ctrlNr,x,y,w,h,label)

obj = getobj(winNr, ctrlNr)

  

Once the slider object handle has been obtained, you can access the slider parameters using the syntax

 

result = obj->parameter()

obj->parameter(value)

 

where parameter is the name of the parameter you wish to access. The first form gets the current parameter value while the second sets the parameter value.

 

Class function list

 

Following is a list of the available slider parameters. Note that the results of these commands are identical to those obtained using the setpar and getpar functions.

 

All parameters must be prefixed with the object reference operator "obj->" where obj is whatever name you have chosen for the control object. As in the above two examples the "set" version requires a value be added as an argument while the "get" version will return the value.

 

value(INT val) .......... gets or sets the value of the slider.

range(VEC range) ........ gets or sets the range of the slider.

orientation(STR dir) .... the visual orientation of the slider.

arrowstep(INT step) ..... the amount to change the slider on press of an arrow key.

pagestep(INT step) ...... the amount to change the slider on press of page up/down.

tickstep(INT step) ...... the distance between ticks on the slider.

 

In addition, all the generic parameters listed in the control class page can applied to the button object.

 

Details

 

value

 

Gets or sets the value of the slider.

 

range

 

Gets or sets the range of the slider. The range is expressed as a two-element vector of [minimum, maximum], such as [0,32] or [-10,10]. By default, a slider's range is [0,100].

 

orientation

 

The visual orientation of the slider in the window. Possible values of dir are "horizontal" and "vertical".

 

arrowstep

 

The absolute amount by which the slider's value changes on receipt of an arrow key while the slider is selected. Defaults to 1.

 

pagestep

 

The absolute amount by which the slider's value changes on receipt of a page up/page down key while the slider is selected. Defaults to 20.

 

tickstep

 

The slider value distance between ticks on its axis. By default, no ticks appear.

 

Example

 

n = window("test",100,100,200,100)

    t = textbox(1,20,20,30)

    s = slider(2,60,20,100,30,"horiz",

          pos = s->value();

          t->text(pos);)

    s->pagestep(10)

    s->tickstep(10)

showwindow(n) 

 

Note that the slider range defaults to 100 unless you invoke the range function with a different value, e.g.

 

s->range([0,50])
