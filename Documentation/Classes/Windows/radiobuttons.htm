The radiobutton control

 

The radio button group control is one of a number of objects which can make up a window. A radio button group is generated using the radiobutton command.

 

Syntax

 

radiobutton(nr,x,y,spacing,orient,states,init,[action]) (eeeeeeec:ddddssss)

 

nr ............ control number (INT)

x,y ........... top left coordinates of control in pixels (INT)

spacing ....... pixels between radio buttons (INT)

orient ........ control direction "horiz" or "vert" (STR)

states ........ names for each radiobutton (STR)

init .......... initial value (which radiobutton) (STR)

action ........ commands to run when control selected

 

Description

 

This command adds a group of radiobuttons to the current window being edited. The radiobutton group can be referred to using unique ID nr. Its location in the window (in pixels) is determined by the x,y integer parameters. The spacing between radiobuttons (in pixels) is determined by the spacing parameter. The radiobuttons may be aligned vertically or horizontally depending on the value of the orient parameter which can have values horiz or vert. Only one radiobutton can be selected at a time and the different possibilities are determined by the comma delimited string list stored in states. The initial choice is determined by the string stored in the parameter init. Finally an optional action may be taken when the radiobutton state is modified.

 

Additional radiobutton parameters can be set or retrieved using the button object handle which can be obtained using either of the following methods:

 

obj = radiobutton (ctrlNr,x,y,spacing,orient,states,init,action)

obj = getobj(winNr, ctrlNr)

  

Once the radiobutton object handle has been obtained, you can access the radiobutton parameters using the syntax

 

result = obj->parameter()

obj->parameter(value)

 

where parameter is the name of the parameter you wish to access. The first form gets the current parameter value while the second sets the parameter value.

 

Class function list

 

Following is a list of the available radiobutton parameters. Note that the results of these commands are identical to those obtained using the setpar and getpar functions.

 

All parameters must be prefixed with the object reference operator "obj->" where obj is whatever name you have chosen for the control object. As in the above two examples the "set" version requires a value be added as an argument while the "get" version will return the value.

 

text() .................. the text associated with the selected radiobutton.

value(INT val) .......... the 1-based index of the selected radiobutton.

init() .................. the 1-based index of the default radiobutton.

option_list() ........... a list of the strings associated with the radiobuttons.

option_string() ......... comma-separated string of the strings associated with the radiobuttons.

orientation(STR dir) .... stacking direction of the radiobutton group.

spacing(INT distance) ... number of pixels between radio buttons.

 

In addition, all the generic parameters listed in the control class page can applied to the button object.

 

Details

 

text

 

The text associated with the selected radiobutton.

 

value

 

The 1-based index of the selected radiobutton.

 

init

 

The 1-based index of the selected radiobutton.

 

option_list

 

Vector of the strings associated with the radiobuttons.

 

option_string

 

Concatenated, comma-separated string of the strings associated with the radiobuttons.

 

orientation

 

The visual orientation of the slider in the window. Possible values of dir are "horizontal" and "vertical".

 

spacing

 

Number of pixels between radio buttons.

 

Example

 

n = window("test",100,100,150,100)

l = statictext(2,20,40,"left","")

r = radiobuttons(1,20,20,20,"horiz","a,b,c,d", "b",

           state = r->text();

           :updateLabel(state))

:updateLabel(r->text())

showwindow(n)

                                                                

procedure(updateLabel,state)

 

 l->text(state)

endproc 

 

Note the form of the command list action. It should be a list of commands separated by semicolons rather than commas. Line-feeds and spaces between commands are ignored.
