The divider control


The movable window divider control is one of a number of objects which can make up a window. A divider is generated using the divider command.

 

Syntax

 

divider(nr,x,y,w,h,orient)         (eeeeee:dvvvvs)

 

nr ............ control number ID (INT)

x,y ........... top left coordinates of control in pixels (INT)

w,h ........... width and height of control in pixels or as a fraction of the window width or height (INT) (note: if the divider is horizontal h = 3, if vertical w = 3)

orient ........ "horiz" or "vert" (STR)

 

Description

 

This command adds a divider to the current GUI window being edited, thereby breaking up the window into two or more regions.

 

Because the divider can be moved it makes no sense to give it a fixed location. Instead the x (for a vertical divider) or y (horizontal) coordinate must be expressed as a fraction of the window width or height (e.g. ww*0.5). This fraction will be modified as the divider is moved.

 

To ensure that other objects move as the divider is moved, it must be attached to a region. A rectangular region is defined by one or more dividers and/or the window boundaries. To assign an object to a region you must use the following command:

 

obj->region(boundaryArray)

 

boundaryArray defines the edges of the region and has the following form

 

[leftObj, rightObj, topObj, bottomObj]

 

The elements of this array are either divider object numbers or window boundaries. The latter are defined as:

 

-1: left window boundary

-2: right window boundary

-3: top window boundary

-4: bottom window boundary

 

For example, consider

 

obj->region([-1, -2, 4, 8])

 

This places object 3 between the left and right window edges. Vertically, object 3 is bounded by divider 4 on top and divider 8 on the bottom. Note that the coordinates of such an object must be expressed in region coordinates. These take their origin at the left window edge (x = 0) and the vertical coordinate of divider 4 (y = 0). These coordinates can be in fixed pixels or relative to the region width (“rw”) or region height (“rh”).

 

Class function list

 

Following is a list of the available divider parameters. Note that the results of these commands are identical to those obtained using the setpar and getpar functions.

 

All parameters must be prefixed with the object reference operator "obj->" where obj is whatever name you have chosen for the control object. As in the above two examples the "set" version requires a value be added as an argument while the "get" version will return the value.

 

region(VEC boundaryArray) ....... sets/gets the region which controls the divider location

 

In addition, all the generic parameters listed in the control class page can applied to the button object.

 

Details

 

region

 

boundaryArray describes the region controlling this divider's location. Please see the discussion under the section "Description", above, for its format and semantics.

 

Example

 

n = window("test", 100, 100, 200, 200, "resizable")

d1 = divider(1,"ww*0.333",0,3,"wh","vert")

d2 = divider(2,"ww*0.666",0,3,"wh","vert")

d3 = divider(3,0,"wh*0.333","rw",3,"horiz")

d4 = divider(4,0,"wh*0.666","rw",3,"horiz")

b1 = button(5,"rw/2-20","rh/2-20", 40, 40, "test")   

d1->region([-1,-2,-3,-4])

d2->region([-1,-2,-3,-4])

d3->region([1,2,-3,-4])

d4->region([1,2,-3,-4])

b1->region([1,2,3,4])

showwindow(n)

 

This places a button at the centre of 4 dividers. When these dividers are moved, the button will be moved so that it remains in the centre of the dividers.
