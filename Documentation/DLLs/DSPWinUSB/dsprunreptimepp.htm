dsprunreptimepp

 

 

Run a pulse program with a DSP controlled repetition time.

 

Syntax

 

dsprunreptimepp([address, [version]])

 

address ... start address on the DSP – default is 0x2000

version ... version number for DSP – default is 1

 

 

Description

 

The simpler dsprunpp command does not include any control over the repetition time. In that case it must be controlled by the PC. However this leads to a small timing jitter of about 1 ms which may be an issue for experiments requiring short repetition times. If you wish to have a constant inter-experiment time then use the command dsprunietimepp.

 

Note that due to USB timing constraints the minimum repetition time is about 35 ms. This is somewhat PC dependent. If you require shorter repetition times considering using the fast mode or writing a pulse program which contains the complete experiment.

 

If you use the normal Kea pulse program compiler then this command will be automatically chosen if you have defined the variable repTime in your pulse program user interface.
