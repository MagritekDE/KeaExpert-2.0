dsprunietimepp

 

 

Run a pulse program with a DSP controlled interexperiment time.

 

Syntax

 

dsprunietimepp([address, [version]])

 

address ... start address on the DSP – default is 0x2000

version ... version number for DSP – default is 1

 

 

Description

 

The simpler dsprunpp command does not include any control over the inter-experiment time. In that case it must be controlled by the PC. However this leads to a large uncertaintly in the interexperiment time because of the various Prospa commands which are not included in the timing loop. This may be an issue for experiments with short inter-experiment delays. If you wish to have a constant repetition time then use the command dsprunreptimepp.

 

Note that the minimum interexperiment time will be macro and PC dependent as it depends on the amount of processing and display Prospa is performing between running scans. If you require shorter repetition times considering using the fast mode or writing a pulse program which contains the complete experiment.

 

If you use the normal Kea pulse program compiler then this command will be automatically chosen if you have defined the variable ieTime in your pulse program user interface.
