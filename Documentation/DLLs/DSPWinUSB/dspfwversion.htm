dspfwversion

 

 

Returns the version number of the DSP firmware code.

 

Syntax

 

ver = dspfwversion()

 

ver ... version number (INT)

 

Description

 

The version number of the DSP firmware gives information about the functionality of the firmware and also which peripherals it expects to be connected. It is a binary code where each bit refers to different functionality or type of peripheral. It also gives information about which version of this firmware is present.

 

bit 0 ... whether repetition or  times can be controlled by the DSP or PC (1) or only on the PC(0) .

 

bit 1 ... original Kea firmware – will normally be set in combination with bit 4 (i.e. ver = 0x0A)

 

bit 2 ... whether fast mode is supported or not.

 

bit 3 ... 4 channel gradient controller is present (this means that the gradient controller will be reset on power-up and DSP reset).

 

bit 4 ... 15 channel gradient controller is present (this means that the gradient controller will be reset on power-up and DSP reset).

 

bit 5 ... 16 channel gradient controller is present (this means that the gradient controller will be reset on power-up and DSP reset).

 

bit 6-15 ... not used

 

Examples

 

ver = 0x0A – original Kea 2 code with support for 4 channel gradient reset.

ver = 0x0D – 4 channel gradient controller reset code with support for DSP and PC repetition time control and fast mode.

ver = 0x15 – 15 channel gradient controller reset code with support for DSP and PC repetition time control and fast mode.

ver = 0x25 – 16 channel gradient controller reset code with support for DSP and PC repetition time control and fast mode.

 

 

The macro DSPFirmware updated in the Kea-Update macro folder should be used to update the firmware version.
