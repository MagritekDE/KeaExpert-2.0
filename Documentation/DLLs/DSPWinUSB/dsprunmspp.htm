dsprunmspp

 

 

Run a multi-scan pulse program on the Kea DSP using fast mode

 

Syntax

 

dsprunmspp()

  

 

Description

 

The pulse program previously loaded into the Kea DSP using the commands dspreadpp and dspwrite will be run. Unlike the simpler dsprunpp command, dsprunmspp runs the pulse program a number of times before exiting. The number of scans is determined by the user interface parameter nrScans. Communcation while scanning is performing using the commands dspsetmspar and dspgetmsdata. The amount of data read is limited to 128 complex data points while 4 parameters can be modified per scan.

 

Because of the way the code is written, the repetition time – which is now controlled by a delay in the pulse program – can be as short as 5 ms (assuming the processing at the PC end is minimal and the computer is reasonably fast).

 

Also because the repetition time is controlled by the DSP timing jitter is negligible (i.e. << 100 ns).

 

 

See also
