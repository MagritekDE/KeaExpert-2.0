User defined commands

 

Prospa commands can come from several locations.  

 

  • Internal or built in commands such as pr, sin, cos etc.
  • DLL commands such as svd, openserial, fieldcalc
  • Class commands such as textCtrl->value, buttonCtrl->label
  • Macro commands such as fft:2d, complex, displayMatrix

 

This is also the order in which commands are searched.

 

However there is a fifth type of command (checked for between class and macro commands) and that is one defined by the user at run time. This is done by assigning the contents of the command to a string which can then be executed e.g.

 

sqr = "procedure(sqr,n); endproc(n^2);"

 

We can now call this procedure using the new name "sqr"

 

> pr sqr(2)

 

   sqr(2) = 4

 

The user defined command can also be used as an alias for a normal command:

 

> func = "sin"

> pr func(pi/2)

 

   func(pi/2) = 1
