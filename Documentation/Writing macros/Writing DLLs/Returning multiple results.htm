Returning multiple results

 

 

Occasionally you may wish to write a command which returns multiple results. The following example (also found in exampleRun.dll) converts a number from cartesian to polar coordinates, returning the magnitude and phase values:

 

/*********************************************************************************

   This procedure converted a number in cartesian coordinates (x,y)

   into polar coordinates (r,theta)

*********************************************************************************/

 

short CartesianToPolar(char *parameters)

{

   short nrArgs;

   float x,y;

   float mag,theta;

  

// Extract arguments

   if((nrArgs = ArgScan(parameters,2,"x,y","ee","ff",&x,&y)) < 0)

     return(nrArgs); 

 

// Calculate polar

   mag = sqrt(x*x + y*y);

   theta = atan(y/x);

 

// Return result to Prospa

   returnVar[1].MakeAndSetFloat(mag);

   returnVar[2].MakeAndSetFloat(theta);

   NumberReturnedValues(2);

  

   return(OK);

}

 

To use this command you can type:

 

> pr ctop(3,4)

 

   5   0.927295

 

or

 

> (m,t) = ctop(3,4)

> pr ("mag = $m$, theta = $t$\n")

       mag = 5, theta = 0.927295

 

 

The only difference between this example and the previous one is that the results are stored in the Prospa global variables returnVar[1] and returnVar[2] instead of ans. If you wish to return more than 2 variables, simply increase the index number i.e. 1, 2, 3 ... (note that it is 1 based not zero as usually the case in C). Up to 50 variables can be returned from a single command.
