Command syntax

 

 

Most prospa commands have a similar form. The user types in the command name followed by a comma delimited list of arguments. (Which are optionally surrounded by parenthesis). i.e.

 

commandName(arg1, arg2, ...)

 

In some cases you may wish to return information to the user. In this case you supply return variable names:

 

var = commandName(arg1, arg2, ...)

 

or

 

(var1,var2, ...) = commandName(arg1, arg2, ...)

 

In most cases the arguments may be mathematical expressions containing other commands e.g.

 

a = myCommand(2*3,sin(23),func(2,3,4))

 

Using the DLL facility you can add commands like these of your own, supplementing the commands built into Prospa. The advantage of this method over using macros is one of speed. (The interpreted macros can be very slow - particularly if loops are used).
