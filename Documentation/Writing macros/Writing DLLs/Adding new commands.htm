Adding new commands

 

 

When Prospa is executed, it dynamically loads in the all DLLs in the DLL folder (or at least those with names of the form *Run.DLL). When a command is being interpreted, the internal command list is first checked and then if the command cannot be found there, each of the DLLs is checked. It does this by calling the AddCommands procedure which must be present in each DLL. If the command still can't be found, then the file system is checked to see if the command is a macro stored on disk.
