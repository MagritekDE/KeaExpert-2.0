Main Menu Help

 

 

The main Prospa menu-bar consists of 3 menus; File, Window and Help, plus a number of definable menus which provide access to useful macros. This manual describes these menus and also discusses how to add additional menus to the menu bar.

 

 

The Prospa main menu

 

The File menu

 

The file menu is divided into several parts. The first two are used for loading and saving matrix data, the third accessing folders, the fourth a miscellaneous collection of commands associated with running macros, the fifth allows predefined window layouts to be loaded and saved while the sixth allows the setting of some important program preferences. The last option exits the program.

 

Load and Save data

 

The Load and Save data commands work with Prospa matrix data. All data sets in Prospa are stored in matrices – which are one, two, three, or (V3) four dimensional. (Although 1D matrices i.e. vectors, are just a special case of 2D matrices).

 

 

 

The Prospa File menu

 

These two commands allow the load and saving of individual matrices to a file. Additional information about using them can be found in the CLI Help manual.

 

Import and Export data

 

The import and export data-menu options allow the loading and saving of data in a variety of numerical formats, including binary and ASCII types. This allows Prospa to communicate with other numerical packages such as Microsoft Excel and Matlab. For more details see the CLI Help manual.

 

 

Open application folder

 

This opens a Windows Explorer interface displaying the contents of the Prospa install directory. This folder is the same as the directory stored in the global variable appdir.

 

Open preferences folder

 

This opens a Windows Explorer interface displaying the contents of the Prospa user preferences directory. This directory is created when Prospa is first run and contains parameters for various dialogs and experiments. If Prospa fails to start it is worth deleting or renaming this folder to see if this fixes the problem.

 

The user preferences folder is the same as the directory stored in the global variable prefdir.

 

 

Set current folder

 

Prospa uses the concept of a current working directory or folder. This is the place where files can be saved and loaded without supplying a directory path. It is the directory listed when you type pwd in the command line interface. Note that this folder is only used for macro commands. All loading and saving via file selection dialogs (e.g. interactively loading a 1D or 2D plot) is done independently of the current working directory. In these cases, each dialog keeps track of its own folder. Changing the current working directory will not change these dialog-specific default directories.

 

When selected, the “Set current folder” option will display the folder browser dialog:

 

 

Simply navigate to the folder you want to have as your current working directory and click on it. Then press the OK button. Note that the current working directory is reset to the application directory (appdir – the directory which holds the Prospa application) at the beginning of each Prospa session. The buttons on the left side of the window allow rapid movement to the Desktop, Computer, the Prospa application directory, the user defined Macros directory and the NMR working directory (as used in Kea and Terranova software).

 

Run a macro file

 

This command presents a file selection dialog from which you can select the macro to load and run. The macro will start executing immediately.

 

 

 

Note that the standard file browser displays the commonly accessed folder in pictorial form, in the same order as in the folder browser.

 

Halt running macro

 

Some macros can be quite time consuming. You can abort these macros either by selecting this option from the menu, or by pressing the escape key.

 

Try running the following macro from the editor and then abort it using this option.

 

# Takes a long time to finish!

for k = 0 to 1e7

  pr k

next k

 

(V3) - In Prospa 3.x this option is only available if the macro is run in the background; however, the escape key still works to stop runaway macros.

 

Save window layout

 

The configuration of Prospa windows' sizes and positions is user definable. Simply move them to the locations you like, and then select this menu option. When the resulting macro is subsequently run, it will restore these positions.

 

Preferences

 

This option displays the Prospa preferences dialog

 

File location

 

All Prospa preferences are stored in the user preferences folder which has the (hidden) location:

 

C:\Documents and Settings\username\Application Data\Prospa\Preferences V3.x

 

in Windows XP, or in Vista and Windows 7:

 

C:\Users\username\AppData\Roaming\Prospa\Preferences V3.x

 

In Prospa this folder is accessible from the main file menu using the first option shown in the above figure or from the command line interface by using the variable prefdir e.g.

 

> cd prefdir

 

When Prospa is run for the first time, the user preferences folder doesn’t exist and predefined preferences stored in:

 

$appdir$\Preferences (typically C:\Program Files\Prospa\Preferences)

 

are copied to the user’s preferences folder. If you have problems starting Prospa you should first try deleting or renaming the Prospa user preferences folder in case files in this folder have been corrupted.

 

Dialog options

 

The second option in the above figure displays the Prospa preferences dialog, which allows the user to modify some of the ways Prospa is displayed and behaves. The preference dialog allows the user to modify and save a number of parameters:

 

  1. Main menus
  2. Macro search path
  3. DLL search path
  4. Working and macro directories
  5. Plot resolution when copying to clipboard or files

Drag and drop option

 

An alternative method for adding new menu items is to drag a drop a folder icon onto the Prospa title bar. A dialog will pop-up asking for the menu name to be associated with this folder and the new menu will then be added. If the folder is inside the Prospa install directory a relative path will be used, while outside this folder an absolute path is taken. This is new method is much faster than the original one if the folder you want to add is visible on the desktop.

 

Resetting the menus to their default configuration

 

If you press the reset button in the Menus tab then the menu will revert to the default configuration and all added menus will be removed. This is useful if you have more than one Prospa version installed and you want to switch between the two.

 

 

 

Saving and Loading menus

 

If you have a menu you would like to save press the Save button and choose a name for the menu. The menu will be stored in the following folder:

 

$prefdir$\StartUp\Menu

 

Similarly if you have saved a menu you can load it in the preferences window by pressing the load button. This automatically updates the Prospa main menu as well.

 

 

Preferences tab – macro search path

 

 

When you access a macro command from the CLI or from another macro you do so by invoking the macro’s name with or without a procedure:

 

testmacro()

 

or

 

testmacro:aProcedure()

 

 

The macro directory is not included in the command. To ensure that the macro can be found, Prospa uses the following search path:

 

  1. The current working directory (as set using the cd() command)
  2. The folder in which the macro containing the command is found.
  3. Any subfolders in folder 2.
  4. The search paths indicated in the preferences macros tab (as shown above).

 

(Note the order change in V3) 

 

In the above example the user-defined search path includes all files and folders inside the Macros folder found under the Prospa application directory i.e. $appdir$\Macros.  This allows macros calls from within Prospa and the supplied menus to access commands stored in the coreMacros folder. (coreMacros is found in $appdir$\Macros\coreMacros)

 

Additional search paths can be appended by pressing the “Add” button and selecting a directory to add using the method described in the user-defined menus preferences section. These might be useful if you add your own macros in a folder elsewhere on your computer.

 

Preferences tab – DLL  search path

 

 

When a DLL command is invoked – for example t2fit(), Prospa will search the path in the order specified in this tab. By default, only one folder is searched - $appdir$\dlls\, however if you write your own DLLs you may wish to place them elsewhere so they don’t get erased accidentally when you upgrade to a new version of Prospa.

 

(V3 The function within the tab option “Plot”, which was available in V2.2, may now be found in the 1D and 2D Pref menu) 

 

Preferences tab – Favourites

 

 

At present this tab controls only two folders – the experiment working directory and the user macro directory. Note that the former is independent of the current directory. These directories are accessible from all file and folder browser dialogs. From within a macro or CLI the working directory may be accessed using the variable "workdir" and the macro directory with "macrodir". If the macrodir includes the variable appdir you will need to use the following command to make use of macrodir

 

cd(evalsubexp(macrodir))

 

Preferences tab – Copy Plot

 

 

When plots a copied a bitmap is placed on the clipboard. By default this will have the same size as the original, however if you require more resolution increase the scale factor in this text box. e.g. 2 will double the resolution in both dimensions. However it will also require four times the memory so take care when copying larger images or plots.

 

Scale factor = 1, image size 100%

Scale factor = 2, image size 50%

 

Note that this factor is also applied when saving plot or images to picture files.

 

The Windows menu

 

The windows menu provides access to each of the default Prospa windows and any additional windows which might be created.

 

The first part of the menu allows any additional windows to be hidden or displayed. This is useful because these windows usually sit on top of the default windows and so can obscure the rest of the interface.

 

The second part of the menu accesses the default windows. These windows are permanent so if you close them they just become hidden and can be redisplayed using this menu. If the window is visible but obscured by other windows it will be brought to the front, unless the obscuring window is a “frontmost” window. The menu items are the titles of the windows. The number at the end of the title are a unique window number and a current subwindow number if the window is divided.

 

Any additional windows present (visible or hidden) will be added to the bottom of the windows menu. These can be displayed individually using these options.

User definable menus

 

As supplied these consist of the following:

 

1D..........  macros for manipulating the 1D plot.

2D..........  macros for manipulating 2D plots.

3D..........  macros for manipulating 3D matrices.

NMRI ...  macros specifically designed for loading and processing NMR imaging data.

NNLS ...  interface to the non-negative least squares command.

GUI .......  macros to design windows.

Layout ...  several  predefined window layouts.

 

Details about these macros can be found in the corresponding help files accessible via the first option in each menu.

 

 

Help menu

 

The help menu provides access to a number of help files which describe different aspects of the Prospa program in some detail. These files are displayed in an HTML viewer which is built into the Prospa application.

 

 

 The “About Prospa” dialog displays a window providing version number, copyright information and contact details to Magritek in case you need some additional help.
