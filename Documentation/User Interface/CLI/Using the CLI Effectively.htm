Using the CLI Effectively

 

 

Recalling previous commands

 

To use the CLI effectively you need to know a few shortcuts. The first is simple. Use the up and down arrow keys to recall previously used commands and then either rerun them by pressing return/enter or edit them and then run them. Typing the first few characters of the command first will only recall commands which started with these characters. Pressing the escape key will clear the current line.

 

The cut (^X) , paste (^V) and copy (^C) functions can also be used to produce new commands in the usual way.

 

Learning the command syntax

 

If you can't recall the argument syntax for a command just type the name of the (valid) command and the required syntax will appear in the CLI status region at the bottom of the window. The status text indicates the type of command (general, class or DLL), the names and types of the different arguments and also the name and type of any returned variables. In this case it shows that there are 3 possible arguments, width, height, depth and hyperdepth (for 4D), all integer and last three are optional. It also shows that the command returns a matrix. You can also select or click on any word in the CLI or text-editor to check the command syntax

 

For more information about the commands or to view a list of the available commands refer to the help menu entry "Help Viewer" in the Prospa menu bar at the top of the program. (Or press F1 when the command name is selected). If you can remember the first character(s) of the command you can type listcom to list all commands starting with this character or characters. e.g.

 

> listcom b

 

   bell

   button

 

Command output

 

If the command normally returns something but you don't supply return variables then these will be printed to the CLI

 

> m = noise(4,3)

> (sx,sy) = size(m)

> size(m)

 

   4   3   1   1

 

The size command returns the dimensions of the matrix m. Note that sizes of all 4 dimensions are returned even if fewer are actually assigned to variables.

 

The print command is optional in the CLI – you can type any expression and the result will be displayed:

 

> 2*sin(pi/4)

 

   2*sin(pi/4) = 1.41421

 

Overly long outputs (try noise(100,100)) can be interrupted by pressing the escape key.

 

Multiple commands

 

Multiple commands can be entered into the CLI by separating them by semicolons. In addition multiple commands can be copy and pasted from the editor onto a single line.

 

> for(k = 0 to 100);  pr k;  next(k)

 

 

Error display

 

If you make a mistake when typing in a command, the CLI will try and explain the nature of the mistake (sometimes a little cryptically!) by printing an error message e.g.

 

> pr "Hi" + 2

 

   Error: 'string + float' is an undefined operation

   Line reads : pr "Hi" + 2

   Error occurred in command #1 of macro : 'CLI'

 

The CLI treats all commands as if they were part of a macro script.

 

Embedding expressions in strings

 

When printing strings you will often wish to include embedded expressions. Do this by surrounding the expression by dollar signs. e.g.

 

> pr ("The answer is : $23*sin(pi/4)$\n")

 

The answer is : 16.2635

 

If you want to control the precision of the numerical result you can add a formatting suffix:

 

> pr ("The answer is : $23*sin(pi/4),2.1f$\n")

 

The answer is : 16.3

 

The syntax used here is the standard formatting used in the C library functions such as printf. There are a number of formats available. These are listed in the appendix A1

 

In addition there are a number of special characters:

 

\n carriage return line feed.

\r carriage return

+- plus minus symbol ±
