Loading and Saving Data

 

 

Using the CLI to load and save files

 

Matrices and lists may be loaded from and saved to disk in native Prospa format using the commands load and save. The syntax for load is:

 

result = load("filename.ext")

 

The contents of filename (which should include an appropriate extension - see below), will be loaded and assigned to the variable "result". Possible extensions are "1d" and "2d" corresponding to 1 and 2 dimensional matrices and "lst" or "par" for text lists. Plots can also be loaded but in this case there is no returned variable. (The extension is .pt1 or .pt2 for 1D or 2D plots respectively).

 

To save a matrix to a file use the command:

 

save("filename.ext",matrix_or_list)

 

Example:

 

t = [0:0.1:100]         # Make a time base

x = sin(t).*exp(-t/10)  # Make a decaying sinusoid

save("fidx.1d",x)       # Save it

y = cos(t).*exp(-t/10)  # Make a decaying cosinusoid

save("fidy.1d",y)       # Save it

a = load("fidx.1d")     # Load x data

b = load("fidy.1d")     # Load y data

plot(a,b)               # Plot it in 1D x,y, mode

 

If you want to save the data to a particular folder or directory then use the cd command first e.g. :

 

cd("D:\Craig's stuff\SOFTWARE\Prospa Feb 20\data")

save("fidy.1d",y)

 

 

Using the menu interface to load and save files

 

An alternative and often more convenient interface to the load and save commands is via the menu interface in the main Prospa window. This menu provides facilities for loading, saving as well as import and export facilities.

 

To load data and/or view data which has been saved in native Prospa format select the "load data" option from the Files menu. The adjacent load/open dialog will appear:

 

By default all files in the current folder of the type *.1d will be visible. If you click on them, then information about the data will be displayed in the lower left corner of the dialog window. For example in this case "fidx.1d" contains complex 1D vector data (since its 2nd, 3rd and 4th dimensions are 1) and has 8192 data points. By default the data variable is called m1d (1d matrix) and will not be displayed when loaded. Check the "Display" box to display the data when it is loaded. Finally press the Open button to load the data set into Prospa. Matrices with higher dimensions can be loaded by selecting the appropriate file type.

 

Saving files works the same way. Select the save menu option to display the save dialog box:

 

 

To use this dialog select the name of the variable to be saved ("x and y" in this case), and then give it a file name ("test.1d"). Finally press the save button. The extension will default to 1d if it not supplied. Matrices with higher dimensions can be saved by selecting the appropriate file type. Note in the case where two variables are saved together (x and y) they should have the same number of elements since they represent the abscissa and ordinate of a graph.
