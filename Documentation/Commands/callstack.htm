callstack

 

Print the current macro call stack to the CLI.

 

Syntax

 

callstack()

 

Description

 

If you want to find out how a Prospa program got to a particular location in a program then add this command and it will print out the various procedure calls required to get to that point. This is a useful debugging tool.

 

Example

 

When running this macro (stored in callTraceTest.mac) the following output will appear on the CLI.

 

1  procedure(callTraceTest)

2

3    :test1()

4

5  endproc()

6

7  procedure(test1)

8

9    :test2()

10

11 endproc()

12

13 procedure(test2)

14

15   callstack()

16

17 endproc()

 

Output

 

--- Stack trace ----

    Macro: 'editorwin.mac', Procedure: 'menu(6,File)', LineNr: 52

      Macro: 'editorwin.mac', Procedure: 'saveAndRunMacro', LineNr: 319

         Macro: 'callTraceTest.mac', Procedure: '', LineNr: 3

            Macro: 'callTraceTest.mac', Procedure: 'test1', LineNr: 9

               Macro: 'callTraceTest.mac', Procedure: 'test2', LineNr: 15

 

This shows that the program started from the Prospa editor via the menu, followed by a call to the procedure saveAndRunMacro. From there it went into callTraceTest.mac on line 3 of the entry procedure (no procedure label in the call) and then to line 9 in procedure ‘test1’ and finally to line 15 in procedure ‘test2’.
