import3dpar

 

Set parameters for 3D import.

 

Syntax

 

import3dpar(parameter, value, parameter, value  ...)

 

parameter ... String chosen from the Parameter column in table below (STR)

value ....... String chosen from the corresponding row in the Value column (STR)

 

Parameter 

 

ab

xyrc

delimiter

fls

machine

 

fileheader

rowheader

 

Value  

 

"ascii"/"binary"

"xydata/"real"/"complex"  

"space"/"tab"/"comma"      

"float"/"double"/"long"/"short"

"win"/"mac"/"bigend"/"littleend"

 

number

number

     

Meaning

 

Text or binary format. Defaults to “ascii”.

Type of data. Defaults to “real”.

Delimiter between data values. Defaults to “space”.

Binary number format. Defaults to “float”.

Little- or big-endian byte order. Defaults to “win”, equivalent to “littleend”.

Number of bytes to skip at start of file.

Number of bytes to skip at start of row.

Description

 

In combination with the import3d command, this routine allows 3D files from sources other that Prospa to be imported. To do this it is first necessary to specify a number of parameters. Which parameters need be specified depends on whether the data set to be loaded is ASCII (text) or binary. In the case of ASCII files it is only necessary to specify the first 3 parameters, while apart from the delimiter all must be specified for binary files. However in some cases the default values may make it unnecessary to specify all parameters (the machine parameter defaults to littleend for example). Note that the parameter - value pairs can be added in any order. For more information see import1dpar.

 

Examples

 

The 3D data set small.3d is stored in the Example Data/Import folder:

 

import3dpar("ab", "binary", "fls", "float", "xyrc", "real",

            "machine", "bigend", "fileheader",0)

n = import3d("small.3d",95,58,30)

image(n[~,~,15])

 

The macro plot3dslices.mac provides an interface allowing different planes or slices from a 3D data set to be viewed.
