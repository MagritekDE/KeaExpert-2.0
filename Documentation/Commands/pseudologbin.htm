pseudologbin

 

Take a large linearly spaced data set and replace it by a smaller pseudo-logarithmically binned one.

 

Syntax

 

(xOut,yOut) = pseudologbin(xIn, yIn, binSize)     (eee:vvf)

 

xIn ....... x axis of linear vector (VECTOR)

yIn ....... y axis of linear vector (VECTOR)

binSize ... number of points in output data set

xOut ...... location of bin centers (VECTOR)

yOut ...... log bin result (VECTOR)

 

Description

 

Some NMR experiments can only collect data with a linearly spacing of results (e.g. CPMG echoes). However inverse Laplace spectral analysis works best when the data is logarithmically spaced. However the first few data points on a log axis will fall between the first points in the linearly spaced input meaning that these points are not unique but rather a linear combination of the first two linearly spaced points.

 

To fix this the pseudologbin command generates an axis that starts out with a linear spacing and morphs into logspacing. In this way each point in the binned data set comes from one or more different points in the linearly spaced data set. You can generate this axis with the command

 

xpLog = pseudovec(xMin,xMax,binSize)

 

where xMin and xMax are the smallest and largest values in xIn.

 

Because of the discrete nature of the data set the resultant xOut values are slightly different from xpLog.

 

 

Example

 

This example generates a biexponential decaying signal (x,y) which has been linearly sampled. It is then log binned to 50 points.

 

x = [0.1:0.1:1000]

y = exp(-x/20) + exp(-x/200)

plot(x,y)

(xl,yl) = pseudologbin(x,y,50)

plot(xl,yl,"symbolshape","circle")

axes("xmapping","log")

title("Pseudo-log binning example")

 

 

 

Note that the first two data points match the spacing of the linear input.
