checkctrlvalues

 

Check that the control values are valid (same as checkcontrols)

 

Syntax

 

result = checkctrlvalues(window_nr, [which, variable])   (eee:dsv)

 

result ...... either "ok" or "error" (STR)

window_nr ... GUI window to check (as returned by window()) (INT)

which ....... which controls to check; either "list" or "all" (the default).

variable .... if which == "list" then variable is the list of control names to consider.

 

Description

 

In some cases a control such as a text box may store a variety of data types. Using the setpar command the allowed types may be restricted to a specified list of types. For example, following the command

 

setpar(0,5,"type","integer")

 

the currently edited window’s control 5 will only allow integer arguments. Further,

    

setpar(0,5, "type","integer","range",[0,10])

 

will only allow integers between 0 and 10 inclusive.

 

checkctrlvalues can be used to check whether the current data contents of a control satisfy all such constraints.

 

This command looks at each specified control in the specified GUI window and checks to see that the current value of that control is of the correct type (integer, float or string) and if it is a number that it falls within the correct range of values. If an error is found the command returns the string "error". On success this command returns the string "ok".

 

Example

 

n = window("test",100,100,150,100)

textbox(1,20,20,80)

showwindow(n)

setpar(n,1,"text","data entry")

checkctrlvalues(n)                      # Succeeds

setpar(n,1, "type","integer")           # Accept only integers in the textbox

checkctrlvalues (n)                     # Fails; contents are non-integer

 

by adding a list of control names (as defined by the setpar parameter “name” or “valueID”) the range of controls checked can be restricted. This is maybe useful in a large interface.
