iscached

 

Check if a macro is cached

 

Syntax

 

(result, [path]) = iscached(macroName, macroPath, [whichCache])

 

macroName ... The macro to search for (STR)

macroPath ... The macro path (use "" if unknown) (STR)

whichCache .. Which cache: "window" (default) or "global" (STR)

result ...... Returns 1 if found, 0 if not found.

path ........ The location of the macro if macroPath is ""

  

Description

 

To reduce the search time for running macros and procedure they can be cached. This is done using the import or cachemacro/cacheproc command. Sometimes it is useful to check if a macro is in the cache (e.g. if the file has been modified outside Prospa and it is in the cache the cached version will be used). There are two caches – one connected to the parent window (the window cache) accessible from all window callbacks and a global cache accessible from anywhere.

 

If the macroPath is not given, then the macro path found is also returned. Note that it is possible to have more than one macro of the same name but with different paths (although this is not recommended).

 

Note that the extension .mac or .pex should be included in the macroName.

 

Also note that the search is not case sensitive.

 

Example

 

This example works in the SpinsolveExpert environment.

 

> pr(iscached("seLock.mac", "","global"))

 

   1   "C:\Users\user_name\Applications\SpinsolveExpert\Macros\Spinsolve-Expert\classes"
