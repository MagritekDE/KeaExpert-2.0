surf2d

 

Draw a 2D matrix as a surface plot.

 

Syntax

 

surf2d(matrix, scale, data_range, scale_range)      (eeee:vvvv)

 

matrix ........ real 2D matrix to render (MAT)

scale ......... colour-scale to use when drawing the surface (MAT)

data_range .... maximum and minimum values in matrix (VEC)

scale_range ... maximum and minimum values in color scale (VEC)

 

Description

 

Draw a surface plot representation of a 2D real matrix.

 

scale determines what colours are used to map data amplitude to colour. It is an n-by-3 RGB matrix (0-1). It may also be one of the predefined colour scales described in cmap.

 

data_range is a two element matrix which determines the vertical extent of the plot. Putting in the correct range will plot the data so that it is scaled to match the x and y axes. For example, if matrix is 100 by 100 points and data_range[1], the maximum, is 50, then the resulting plot will be twice as wide as it is high.

 

scale_range is a two element matrix which determines how the data is mapped to the colour scale.

 

For a normal colour scale, the first element of scale_range maps to the bottom value in the colour scale, while the second element maps to the top value.

 

If scale is of the +/- kind, then both elements of scale_range should be positive, with the first element specifying the 0 level (all data values between this number and zero will be mapped to the zero colour), while the second element determines the data value which is mapped to the maximum colour value. For negative numbers the same interpretation applies, but with the sign reversed.

 

Example

 

x = [0:1:100]

x = sin(2*pi*x/40).*exp(-(x-50)^2/1000)

m = x'*x

image(m)

draw3d("false")

clear3d()

shift3d(-50,-50,0)

surf2d(m,spectrum,[-1,1],[0,0.8])

scale3d(1,1,40)

rotate3d(40,0,0)

draw3d("true")

for(r = 0 to 360)

   rotate3d(40,0,r)

   pause (0.05)

next(r)

 

See cmap for a list of predefined colour scales.

  

See also
