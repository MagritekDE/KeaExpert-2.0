fixlist

 

Escapes any unescaped quotes in a list

 

Syntax

 

result = fixlist(list, [mode])

 

list ..... the list to be checked

mode ..... what to fix – currently this defaults to "quotes"

result ... the list after being ‘fixed’

 

Description

 

Normally all strings embedded in a list should be escaped. However if loaded from a parameter file this may not be the case.

 

Example

 

Here are the contents of a file called test.par

 

a = 23

b = [1:10]

c = "This is an "embedded" string"

 

If we load this into Prospa:

 

> par = load("test.par")

> pr par

 

   par = {

       a = 23

       b = [1:10]

       c = "This is an "embedded" string"

   }

 

This list cannot be fully used because of the unescaped string in parameter c. However, the fixlist command can solve this problem:

 

> pr fixlist(par)

 

   fixlist(par) = {

       a = 23

       b = [1:10]

       c = "This is an \"embedded\" string"

   }
