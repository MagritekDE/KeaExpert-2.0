threadabort

 

Abort a running procedure thread.

 

Syntax

 

result = threadabort(id)    (e:d)

 

id .......... thread id – obtained from the thread command.     (INT)

result ...... 1 if thread stopped, 0 if not running to start with.

 

Description

 

This function stops a thread from running.

 

id can refer to the current thread, in which case this command is equivalent to abort.

 

Example

 

procedure(threadTest)

 

   n = window("Thread test",-1,-1,130,30)

      statictext(1,10,10,"left","0")

      windowvar(id)

   showwindow(n)

   id = thread(":timer")

   setwindowpar(n,"exit_procedure","threadabort(id)")

 

endproc()

 

procedure(timer)

 

  oldt = ""

  while(1)

     t = gettime()

     if(t != oldt)

        setpar(0,1,"text","The time is: $t$")

        oldt = t

     endif

  endwhile()

 

endproc()

 

This macro will display a small window in which the time is displayed. While this window displays, you can move it around and use the rest of Prospa without restriction.

 

When you close the window, the threadabort command is used to stop the process. Note that you must abort the thread before closing the window. Failing to abort the thread before closing the window may crash Prospa.

 

The pause command is useful if you expect to run multiple threads. If you don't include pause and attempt to run other threads then Prospa will freeze up; the first thread will not give the other threads any time to run. You will also notice that the CPU usage by Prospa will rapidly increase to almost 100%.
