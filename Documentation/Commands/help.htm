help

 

Open a Prospa HTML help file found in the documentation folder.

 

Syntax

 

help()

 

help(command)                       (e:s)

 

help(path, filename, [bookmark]) (eee:sss)

 

command ...... name of internal or DLL command. (STR)

filename ..... filename and path within the documentation folder. (STR)

path ......... path to help file within the documentation folder. (STR)

bookmark ..... book mark within file to open. (STR)

 

Description

 

The first version of this command can be used to open the help viewer to a help page giving a general introduction to Prospa. Help on more specific aspects of Prospa can be reached through a navigation pane within the help window.

 

The second version of this command can be used to obtain help on specific Prospa commands – either internal commands or commands added using DLLs. All help files should be stored in the Prospa documentation folder. Within this folder are several sub-folders, one of which is called "DLLs", and the other "Internal". If a new DLL is added then a sub-folder within DLLs should be added and the help for those DLL commands placed in that folder. For example, all fitting DLLs commands are stored in "$appdir$\Documentation\DLLs\Fitting".

 

The third version of the command is lower level, since it requires that you enter an absolute path or one relative to the Documentation folder to specify where the help file is stored. A book mark can also be included with this option.

 

Note that the “htm” extension is required in filename.

 

Examples

 

help("biexpfit")

help("DLLs\\Fitting","biexpfit.htm")
