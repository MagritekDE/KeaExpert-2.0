zerofill

 

Add zeros to a 1D data set to increase its size.

 

Syntax

 

result = zerofill(vector,size,mode) [eeee:vlsf]

 

or

 

result = fill(vector,size,mode,value) [eeee:vlsv]

 

vector ... vector to zerofill (VEC/CVEC)

size ..... new size of vector after zerofilling (INT)

mode ..... where to add zeros (“start”|”sides”|”end” (STR)

result ... new vector with zeros added. (VEC/CVEC)

value .... number to fill data set with (FLOAT/COMPLEX)

 

Description

 

This command can be used to extend the size of a vector by adding zeros (or some other number) to the beginning (“start”), end (“end”) or both sides (“sides”). In the case of both sides the number of zeros will be equally split between the two.

 

zerofill is typically used to force a data set to be a power of 2 in length so that an FFT can be applied. It is also used for interpolating a data set, as shown in the example given below.

 

Example

 

x = [0:1:255]/10

y = exp((i-0.1)*x)

fy = ft(y)

zffy = zerofill(fy,16384,"sides")

zfy = 16384/256*ift(zffy)

draw1d("false")

multiplot("1d",1,2)

curplot("1d",1,1)

plot(y)

title("Original (256 points)")

axes("fontsize",8)

xlabel("")

curplot("1d",1,2)

plot(zfy)

title("Zero filled (16384 points)")

axes("fontsize",8)

xlabel("")

draw1d("true")

 

In this case the small artefacts arise because the Fourier transformed data had not dropped to zero at the ends of the data set, giving rise to a step in the frequency domain data.
