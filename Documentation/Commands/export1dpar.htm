export1dpar

 

Specify parameters to be used in the export1d command.

 

Syntax

 

export1dpar(parameter, value, parameter, value ...)

 

parameter ... String chosen from the Parameter column in table below (STR)

value ....... String chosen from the corresponding row in the Value column (STR)

 

Parameter 

 

ab

xyrc

delimiter

fls

machine

 

fileheader

 

Value  

 

"ascii"/"binary"

"xydata/"real"/"complex"  

"space"/"tab"/"comma"      

"float"/"double"/"long"/"short"

"win"/"mac"/"bigend"/"littleend"

 

number

            

Meaning

 

Text or binary format. Defaults to “ascii”.

Type of data. Defaults to “real”.

Delimiter between data values. Defaults to “space”.

Binary number format. Defaults to “float”.

Little- or big-endian byte order. Defaults to “win”, equivalent to “littleend”.

Number of bytes to skip at start of file.

 

 

Description

 

In combination with the export1d command this routine allows 1D files to be exported from Prospa in user-specified formats. To do this it is first necessary to specify a number of parameters. Which parameters need be specified depends on whether the data set to be loaded is ASCII (text) or binary. In the case of ASCII files it is only necessary to specify the first 3 parameters, while apart from the delimiter must all be specified for binary files. However in some cases the default values may make it unnecessary to specify all parameters (the machine byte order defaults to littleend for example). Note that the parameter - value pairs can be added in any order.

 

ASCII file formats

 

In text files, data may be delimited by spaces, commas, tabs or returns. For example the following file formats are seen as the same and may be imported as either ydata or xydata. Files from machines which use different line endings (i.e. Macintosh, Unix or Windows) will all be treated the same.

 

1,2

3,4

5,6

 

1,2,3,4,5,6

 

Binary file format

 

Currently, 4 number formats are supported:

 

float – 4 byte signed floating point format

double – 8 byte signed floating point format

long  – 4 byte signed integer format

short – 2 byte signed integer format

 

Since different computers store their data in different byte orders it is necessary to have an option to allow for this. Macintosh computers (and most other RISC based machines machines such as SGIs or SUNs) store their data with the most significant byte in lower memory (big-endian) while Intel (Windows) based machines (and a few others) use the reverse format (little-endian). If you are not sure which is correct, try both. If you've chosen incorrectly the data will have crazy values. The default option is the Windows format littleend. 

 

Example

 

x = [0:1:1023]/10

y = exp((i-0.2)*x)

y = ft(y)

export1dpar("ab", "binary",  "xyrc", "complex",

           "fls", "float", "machine", "bigend",

           "fileheader",0)

export1d(y, "spectrum.dat")

 

import1dpar("ab", "binary",  "xyrc", "complex",

           "fls", "float", "machine", "bigend",

           "fileheader",0)

n = import1d("spectrum.dat",1024)

plot(n)
