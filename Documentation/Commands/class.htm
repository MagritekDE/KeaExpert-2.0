class

 

Generate an object from a class

 

Syntax

 

result = class(class_name::init)

 

result ...  Instantiated object (CLASS)

 

 

Description

 

This command implements a very basic class which can be used for organising data and procedures. It does not support inheritance or other more sophisticated features found in other languages and is purely introduced to simplify data access.

 

Note the difference between class and object. The class is the template which defines the object. An object is simply one version (an instantiation) of the class. The contents of each object can be quite different, but they all share the same procedures and many of the same member variables.

 

Classes are an extension of the structure data type; adding procedure references and an initialiser (a constructor).

 

Before the class data type was introduced the only way to pass variables to procedures in Prospa was via the argument list or global variables (or window variables). This can result in excessively long argument lists or the need to use string lists to pack the necessary arguments. When a class is used, it is possible to have variables (members) which are part of the class and which can be accessed directly from a class procedure. Therefore they don’t need to be passed as arguments as in simple procedures.

 

Before the class command can be called it is necessary to import the class macro. The syntax is:

 

   import(class_macro, macro_path, scope)

 

This is equivalent to caching a macro. scope can be "global", "window" or "local" and defaults to "global". This means that you can access the class from anywhere in the Prospa environment.  Window means you can access it from anywhere in the current window, while local means you can only access if from the current procedure.

 

Then we can call the class command

 

  objectName = class(class_name:init)

 

Note that the class_name is just the class macro name without the .mac extension (which is optional for class_macro). In this case objectName is only accessible locally. To make it a global or window variable use the assign command.

 

assign("gName", class("class:init"), "global")

assign("wName", class("class:init"), "window")

 

here gName is the globally accessible object name and wName is the window accessible object name.

 

What the class command does is generate a structure called objectName. It initialises this structure with the members stored in the init procedure found in the class macro file. The format of this procedure is:

 

procedure(init, self, arg1, arg2 ... argN)

 

   self->procedure1 =  "macro_name:procedure1"

   self->procedure2 =  "macro_name:procedure2"

      ...

   self->procedureN =  "macro_name:procedureN"

 

   self->member1 =  member1

   self->member2 =  member2

      ...

   self->memberN =  memberN

 

endproc() 

 

the variable self here is a reference to the object you are creating. argX are optional arguments.

 

Note that class procedures should normally be stored in the class macro so macro_name == class_name.

 

Once the class command has returned, you can access the class procedures or members using the arrow syntax

 

objectName->procedure1(argument_list)

 

objectName->member1

 

The only thing special about the class procedures is that they have ‘self’ as the first argument, allowing access to the object member variables in addition to the normal arguments e.g.

 

procedure(procedure1, self, arg1, arg2, ... argN)

 

   a = self->member1

 

endproc()

 

Effectively we have extended the local variables in a procedure to include all those variables stored in the object. These variables will also be accessible to all other member procedures. No copying required.

 

Note that member variables can be added to the object outside the class file.

 

Accessing class procedures.

 

To allow rapid text editor access to class procedures in the same way as normal Prospa procedures, you need register the names you are using for the classes to Prospa. Do this with the classinfo command once the classes have been defined e.g.

 

   assoc = ["quick   = parabolic",

            "power   = simplex"]

 

   classinfo(assoc)

 

This associates the class instance 'quick' to the 'parabolic' class macro and 'power' to the 'simplex' class macro. Once you have done this it means that if you want to jump into a class procedure in 'quick' you can control double-click on the procedure name as for normal procedures.

Important note: classes are passed by value

 

If you pass the object to another procedure (not a class procedure) it will not be able to modify the object content permanently since the object will be copied when passed as an argument (this is true for all Prospa variables except built in classes like plots). You either need to return the object from the procedure and overwrite the original object or pass the object as an alias.

 

myObject = class(class_name)

myObject  = :test_procedure(myObject) # Test procedure modifies member variables in myObject

:test_procedure(alias(myObject))      # Equivalent and more efficient way to do this

 

Passing by alias is the same as passing by reference in other languages.

 

 

 

Example 1

 

This example creates a simple class to determine the area and circumference of a circle. The chief difference with the procedural way of solving this problem is that we are asking the object to supply this information rather than sending the radius to a dedicated area or circumference procedure.  In this simple example the advantages of using an object may not be clear, but when there are many variables and procedures the class approach is much tidier and more understandable.

 

Class file "circle.mac"

 

procedure(circleTest)

 

   import("circle.mac", getmacropath())

 

   myCircle1 = class("circle:init",1) # Define a circle object with radius 1

   myCircle2 = class("circle:init",2) # Define a circle object with radius 2

 

   print(myCircle1->area())

   print(myCircle1->circumference())

 

   print(myCircle2->area())

   print(myCircle2->circumference())

 

endproc()

 

  # Initialise the circle class

procedure(init, self, radius)

 

   self->area = "circle:area"

   self->circumference = "circle:circumference"

 

   self->radius = radius # Default radius

 

endproc()

 

# Calculate the circle area

procedure(area, self)

 

  a = pi*self->radius^2

 

endproc(a)

 

# Calculate the circle circumference

procedure(circumference, self)

 

  c = 2*pi*self->radius

 

endproc(c)

 

Here is the print output

 

myCircle1->area() = 3.14159

myCircle1->circumference() = 6.28319

 

myCircle2->area() = 12.5664

myCircle2->circumference() = 12.5664
