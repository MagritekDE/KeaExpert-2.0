Replacing an incorrect Terranova driver on a 32 bit XP computer

 

 

When installing the driver for the Terranova on a 32 bit XP computer it is possible to install the 64 bit version from the CD by mistake. The result is an error message when you try and communicate with the device using Prospa. This document shows how to resolve this issue.

 

First connect your computer to the Terranova with a USB cable (< 3 m long and connect to a rear port) and then switch it on. Open the System Properties dialog (right click on the “My Computer” icon), select the Hardware tab and then press the Device Manager button.

 


 

 Then check under the USB device list that there is a device called USBDSP.

 

 

This is the correct driver for a 32 bit XP system and you need to look elsewhere for a solution to the connection problem.

 

If instead you see a device called “DSP usb driver” then you have somehow managed to install the 64 bit version of the driver. 

 

If it does look like you have the wrong driver installed then follow these steps to change the driver

 

With the Terranova connected to the computer and switched on, open the device manager and uninstall the driver (you may also be able to do this using the Update Driver option).

 

 

Next cycle the power on the spectrometer and when the Found New Hardware Wizard comes up select "No, not this time"

 

 

 

and then "Next" and then "install from a list or specific location". Press Next ...

 

 

and then "Don't search. I will choose the driver to install." Press Next

 

 

And then press "Have Disk ..."

 

 

Browse to the USB Driver folder in the Prospa install directory or the 32 bit installer on the supplied CD and select magritekUSB.inf:

 

 

 

(The 64 bit version is called DSPUSB.inf). Press Open and then OK in the next dialog (your file path will be different):

 

 

 

Finally press "Next" once the USBDSP shows up in the compatible hardware list

 

 

And in the final dialog (not shown) press "continue installation" to complete.
