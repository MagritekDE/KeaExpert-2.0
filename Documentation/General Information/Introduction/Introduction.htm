Prospa V3.xx - Introduction

 

 

Prospa is a software package designed specifically for the processing of NMR experiments. With additional hardware it can also be used to control an NMR experiment and collect the resultant data. However in this introduction we will concentrate on the processing of NMR data.

 

Prospa is based around a core of 400 or so commands which can be used to load, save, modify and view numerical data in matrix form. These commands can be combined together to form sophisticated script files called macros which can perform a wide variety of processing and interface functions.

 

Prospa Window Organization

 

The default Prospa configuration consists of 6 main windows. The main Prospa window acts as a container for the other 5 windows and provides a main menu. Resizing this window will cause all windows inside it to be resized as well. The main menu contains options for loading, saving, importing and exporting data files, and running macro files.

 

 

The Prospa window showing (counter-clockwise from top), the 1D plot, the 2D plot, the CLI, the editor and the 3D plot.

 

The other 5 windows consist of:

 

  1. A command line interface (CLI), into which typed, single-line commands can be entered. This also provides an output interface for diagnostic message from other parts of the program.

 

  1. An editor into which macros can be entered, loaded, modified and saved. The editor provides syntax highlighting and command help to simplify the writing of macros. The editor may also be subdivided to edit multiple files. Additional editors can also be opened.

 

  1. A 1D plot into which x-y type data can be displayed. It provides facilities to display real and complex data, error bars and multiple traces. Like the editor the 1D plot can be subdivided to display multiple data sets and additional plots may be opened.

 

  1. A 2D plot into which images can be displayed. They may be in the form of intensity or contour plots. Rows and columns from the current 2D data set can be easily displayed in the 1D plot. Again 2D subplots can be formed for displaying multiple images.

 

  1. The 3D plot window can display a variety of plot types – for example a surface rendering of the data in the current 2D plot, an iso-surface rendering of a 3D data set and 3D lines and surfaces. Interactively selected planes from the 3D plot may also be viewed in the 2D plot. Although the 3D plot can't be divided like the 1D and 2D plot multiple 3D  plot windows can be displayed.

 

All 5 subwindows provide their own menus and shortcuts.

 

As well as the 6 main windows, Prospa also allows the end-user to design their own windows. These are typically used as control panels for collecting, processing or viewing data. In the Prospa manuals these are referred to as graphical user interface, or GUI windows, since they provide a more convenient interface between the user and the program than raw macro commands.

 

 

A custom interface for controlling the display of mulitple 1D plots.

 

Although Prospa is supplied with the above interface as the defaut all aspects of the Prospa interface are user definable and several alternative interfaces are supplied under the "layout" menu.

 

Prospa Main Menus

 

In addition to accessing data files and the various windows displayed by Prospa the main menu includes a number of user definable macro menus.

 

  • The 1D and 2D menus contain macros which act on the currently displayed 1D and 2D data sets. 

 

  • The 3D menu acts on 3D matrices currently loaded in to the program and displays them in the 2D or 3D plot windows.

 

  • The NMRI menu – discussed in more detail below –  is specifically designed for processing NMR and NMR Imaging data sets.

 

  • The GUI menu includes macros which can be used for the design and debugging of new interface windows.

 

  • The Layout menu contains several macros which are useful for organizing the Prospa window layout.

 

  • Finally, the Help menu contains links to HTML help files for the various parts of the program – including this file.

 

Other menus may be present depending on the options you have purchased with Prospa.

 

Note that most menus have an associated help file which is accessible via the first menu entry.

 

Loading NMR data sets – an introduction to using Prospa

 

Prospa NMR data sets consist of a single data file in binary or ASCII (text) format and an associated (human readable) parameter file. This contains all the information necessary to load the data file correctly.

 

Because each spectrometer manufacturer uses their own propriety data format it is necessary for Prospa to know this format before it can load and display the NMR data. It does this by reading the manufacture’s parameters file (or in some cases a binary data file header) before reading the data itself.

 

 

A macro called importData has been written to perform these functions. This macro can be found in the main file menu under the NMR imaging menu (NMRI). A screenshot of this macro is shown below. The working directory at the top is the location where you have saved your NMR data sets.

This directory will be remembered from session to session. The list on the left represents all the folders in the current path (which initially will be the same as the working directory.) Those with icons indicate that there is a readable data set inside the folder.

 

 

The importData macro being used to load a k-space NMR data set

 

These data sets should have been copied directly from the spectrometer – either with the ftpDownload macro in the NMRI menu or using external software. In addition to the data file itself, this folder must also contain the spectrometer produced parameter files.

 

If you single-click on a data set folder in the left-hand list the parameters will be loaded and displayed. If you double-click on a folder, the data set will be loaded and (if the Display flag is checked), displayed as well. Alternatively you can single-click and then press the load button. This method has the advantage of allowing some of the parameters to be modified before loading.

 

After being loaded a first time, a new parameter file called exp.par will be stored in the data folder. This contains all the necessary information for Prospa to load the data set in an efficient way in the future. If you want to load the data set using the original parameter file then select the "Load using native parameters" checkbox option .

Fourier transforming

 

Once image data has been loaded the next step is often to transform it from the k-space domain in which it is acquired, to real space. This can be done using the nDFourierTransform macro, also found under the NMRI menu.

 

 

If the kumquat dataset was the last loaded, then this will appear as the default data set in this macro. The dimension, name and size of this data set will be reported at the top of the window.

 

Once the correct data set has been displayed, you will need to enter the processing parameters. (If the correct data set isn’t displayed, then first choose it using the dimension and matrix name text-menus at the top left of the window). If you have loaded the example data set these will be present already, having been entered into the exp.par file, otherwise choose the output matrix name, dimensions of the final data set (which should be greater than or equal to the original, and a power of 2), any k-space filtering you wish to apply and the type of transform to be applied in this case FTEcho. (Which is appropriate for k-space data sets which peak in the centre.)

Viewing and analyzing the resultant data set

 

Once displayed, the 2D image may be post-processed in a variety of ways depending on your application. All the macros in the 2D main menu can be applied to this data set and slices from this data set may be analyzed using the 1D menu option. Access the 1D or 2D Macro Help options in the corresponding menus for more details

 

Saving or reprocessing your data

 

At this stage you can save the processed data using the save plot option in the 2D plot file menu (see adjacent figure). This will save the visual information in the 2D plot along with the matrix data it is derived from. Alternatively you can save the raw 2D matrix data using the save-data option in the main Prospa file menu. (The matrix will be called kumquatFT). However the latter option will not save the colour scale or axes preferences.

2D-Plot File menu

 

An alternative to saving the processed data is to reprocess it. This is useful if the processed data set takes up a lot of disk space. To reprocess efficiently we need to save the processing parameters. This can be done by running the NMRI macro saveParameters.

 

This adds the new processing parameters (which in this case have been stored with the matrix kumquatFt) to the exp.par file so that next time you process then you don’t have to remember the parameters you have previously chosen. (This has been done in the above example). For details on this process and other NMRI menu options select the NMRHelp option from the NMRI menu or press the Help button in the appropriate GUI window.

 

 

Further processing using the NMRI macros

 

At present the NMR menu provides options to FTP data from a spectrometer, import this data into Prospa, forward and inverse Fourier transform it and then further process it to produce relaxation, diffusion and velocity data weighted images.  For details of these functions please consult the NMRI Help menu option.
