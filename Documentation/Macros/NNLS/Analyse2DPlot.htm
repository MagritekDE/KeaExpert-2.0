NNLS – Analyse2DPlot

 

This macro provides an interface for inverting 2D data sets using non-negative least squares (NNLS) analysis. Data set should have been collected using either the T1T2 or DT2 macros. These experiments are available for the Kea or Rock Core Analyser spectrometers.

 

 

 

 

 

 

 

 NNLS   Inversion

 

 

The inversion process

 

The macro generates the following user interface which allows selection of inversion parameters along with display of the inversion, inversion statistics (deviation) and the selection of a suitable smoothing parameter. Inversions can be performed using the algorithm of Lawson and Hanson (reference 1), with 2D extensions based on the ideas of reference 2, or using Maximum Entropy using the method described in reference 3.

 

The graphical user interface for the 2D inverse Laplace calculation

 

Input data should be in Prospa 2D plot format (*.pt2) and can be selected using the file selection button (labeled “...” top middle of interface) or alternatively it can work on the currently selected 2D data.

 

 

In the latter case the plot will be fixed to the current 2D plot at the time the macro was started. The plot being used can identified by pressing the Show button (this will toggle the color scale on that plot). A new current plot can be chosen by selecting the plot and then pressing the Update button.

 

Input parameters

 

Once the data set has been selected you will need to give the input parameters for the correlation spectrum.

 

 

This includes:

 

·         The algorithm to use. L+H = Lawson and Hanson, M.E. = Maximum entropy

·         The type of experiment; T1T2 or DT2

·         The time units; seconds or milliseconds.

·         Whether log-binning should be used on either the x axis or y axis. This is only relevant if the axis has been linearly sampled and is not the y-axis of a DT2 experiment. Only valid options are enabled. T1T2 and DT2 experiments should use x-axis log binning. See below for comments on when to use log-binning.

·         The diffusion parameters (if DT2 selected). This is the gradient duration (little delta), the gradient separation (big delta) and the ramp time of the pulses. All units here are in miliseconds. Currently the inversion only supports diffusion data obtained by varying the gradient amplitude.

Smoothing the inversion

 

The inversion process does not have a unique solution. In most cases the best fit to the original data will occur when many isolated peaks are present in the spectrum. However the noise in the original data set means that such high quality fits are not justified and at best the residuals (difference between original data and simulated data from the inversion) should have a variance (square of standard deviation) no smaller than the variance in the original data set. We can adjust this variance by introducing a smoothing factor which broadens the spectral peaks as it becomes larger.  An example of the effect of changing the smoothing factor is shown below in which the smoothing factor has been set to 0.001, 0.01, 0.1, 1.

This is typical of the shape of the smoothing curve: starting out at a plateau with a  close to ideal error and then rising as the smoothing factor is increased. The indicated point is the so called “knee” in the smoothing curve which occurs when the slope of the log-log variance vs smoothing plot is 0.1. This is a compromise between being overly smoothed and undersmoothed.

If the optimal point is not visible the you will need to increase the range of the smoothing curve. Noisy data sets (with SNR < 100) will require larger smoothing values.

 

Once an optimal smoothing value has been found a larger spectrum can be calculated. Note that the calculation time for the Lawson and Hanson method increases rapidly with spectrum dimension e.g. a 20 by 20  inversion of the above data set to 0.6 s while a 40 by 40 data set took 55 seconds. The largest spectrum which could be calculated with the Lawson and Hanson method for this data set was 80 by 80 and this took 2747 seconds and required ~1 GBytes of RAM! Note that the inversion time increases as more features are present in the spectrum and consequently also increases for larger smoothing factors.

 

 

 

The maximum entropy (M.E.) method gives a similar result with an optimal smoothing factor of 0.228

 

 

In this case the calculation time is much quicker for larger spectra: 2.3 seconds for a 40 by 40 result and 45 seconds for 80 by 80. The chief disadvantage of the M.E. method is the inability to converge for very small smoothing factors. Smaller smoothing factors also result in longer inversion times (the reverse of L&H).

When to use log-binning

 

The Lawson and Hanson algorithm compresses the data using singular value decomposition and so there is only a slight speed improvement to be gained by first log binning the data, although memory consumption will be reduced. In the case of the Maximum entropy method there are considerable performance gains to be made by log binning the x-axis. A bin size of 50 is usually adequate.

 

The macro can also process data sets with log-log, log-linear or linear-linear axes. In all cases the source data set should be suitably calibrated and displayed so the macro knows how the data has been collected.

Peak integration

 

Peaks in the inversion can be integrated to see what fraction of the total signal intensity they contain. To do this select the option “select a rectangular region” in the view menu and then draw a rectangle around the peak of interest

 

 

Then select the integrate peak option from the Calculate menu

 

The peak integral and location will then be reported in the status bar and in the command line interface

Known issues with the software

 

Here are some known problems or limitations with the software which may restrict its operation when used with non-Magritek acquired data sets or which otherwise give non-ideal results.

 

 

1.      The DT2 inversion currently only supports experiments which vary the gradient linearly (the only type available in the Kea/RCA pulse programs).

2.      If the axis type or units are incorrectly chosen the inversion may fail and no longer be accessible because all controls are disabled. Just close the window and start it up again.

3.      The Lawson and Hanson algorithm is very slow for larger output spectra. Spectra should be limited to less than 50 by 50 or even 30 by 30 if large features are expected. The Maximum Entropy algorithm can generate data sets up to 100 by 100 points in reasonable times. Always start with small spectra sizes (15 by 15) before proceeding to larger ones.

4.      The Maximum Entropy algorithm may fail to converge for small smoothing factors. Observe output on the command line interface for an indication that this has occurred (dots will appear in the output).

 

These limitations will be addressed in future versions of the software.

Additions to the software Sept 2017

 

Since this document was originally written there have been a number of improvements made to the software which address the limitations mentioned above

 

  1. A new inversion method has been added based on the FISTA algorithm4. This is much faster than L&H or Maximum Entropy, always converges and scales linearly in time with output matrix dimension. This allows output data sets greater than 100 by 100 pixels to be calculated in a short time.

 

  1. New 2D experiments added:

T2T2 : kernel =>  exp(-tx/T2x) exp(-ty/T2y)

T1SRT2 : kernel =>  exp(-tx/T2) (1-exp(-ty/T1)

DT2 (PGSE-CPMG) : kernel  =>  exp(-tx/T2) exp(-D[g2G2d2 [D-d/3]]y)

DT2 (SGSTE-CPMG)-D : kernel  =>  exp(-tx/T2) exp(-Dg2G2d2 [Dy+2d/3])

DT2 (SGSTE-CPMG)-d : kernel  =>  exp(-tx/T2) exp(-Dg2G2dy2 [D+2dy/3])

 

  1. User defined axes can be supplied for 2D experiments. Just save the axes information in a text file with each time step on a separate line.
