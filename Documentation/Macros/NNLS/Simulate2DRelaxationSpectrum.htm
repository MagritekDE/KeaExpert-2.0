NNLS – Simuate2DRelaxationSpectrum

 

This macro provides an interface for simulated 2D relaxation spectra with multiple peaks, with different shapes, positions and amplitudes. Then from this spectrum simulated data is created with additional noise added if desired.

 

The user interface

 

The interface is divided into 4 parts:

 

·         On the top left we choose the number of peaks to appear in the spectrum and how many data points there should be. The relaxation time range (in milliseconds) can also be entered.

 

·         In the central region are the control buttons to generate the spectrum, populate the peak parameters, load and save data, display this help file and exit the window.

 

·         In the bottom region are the parameters for each peak. For each peak we can specify the position (a number from 0 to 1), the width and height (0-1), rotation angle (in degrees), amplitude and peak type.

 

·         In the top right region are the parameters for the data set which is obtained from the spectral data.

 

When the number of peaks is modified from the pull down menu ‘Number of Peaks’ a corresponding number of peak parameters will be displayed at the bottom of the window, the window resizing to accommodate them. You can automatically populate the parameter text fields using the ‘Auto Fill’ button in the central part of the window. This produces N equally spaced Gaussian peaks with the same width. The parameters for the simulated data generated from this spectrum include the number of points, the minimum and maximum times, the amplitude of the white noise added to the spectrum and the spacing of the data points (logarithmic or linear). The smoothing parameter at which the (logged) slope is 0.1 is indicated by the solid square and is at 0.034. In the Output parameters section enter this into the ‘Smoothing’ text box and press the ‘Calculate’ button. Here are the results for a 20 by 20 output matrix and a 50 by 50.

 

 

For the Lawson and Hanson algorithm there is a big difference in the processing time, in this example 3 sec for 20 by 20 and 3403 sec for 50 by 50!  With the maximum entropy method very similar results can be obtained in only 52 s however the convergence for small smoothing factors is often not as reliable using this method. However because of the shorter processing times it is always worth trying maximum entropy first.
