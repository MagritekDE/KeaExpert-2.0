NNLS – Simulate1DRelaxationSpectrum

 

This macro provides an interface for simulated 1D relaxation spectra with multiple peaks, with different shapes, positions and amplitudes. Then from this spectrum simulated data is created with additional noise added if desired.

 

The user interface

 

The interface is divided into 4 parts:

 

·         On the top left we choose the number of peaks to appear in the spectrum and how many data points there should be. The relaxation time range (in milliseconds) can also be entered.

 

·         In the central region are the control buttons to generate the spectrum, populate the peak parameters, load and save data, display this help file and exit the window.

 

·         In the bottom region are the parameters for each peak.

 

·         In the top right region are the parameters for the data set which is obtained from the spectral data.

 

When the number of peaks is modified from the pull down menu ‘Number of Peaks’ a corresponding number of peak parameters will be displayed at the bottom of the window, the window resizing to accommodate them. You can automatically populate the parameter text fields using the ‘Auto Fill’ button in the central part of the window. This produces N equally spaced Gaussian peaks with the same width. The parameters for the simulated data generated from this spectrum include the number of points, the minimum and maximum times, the amplitude of the white noise added to the spectrum and the spacing of the data points (logarithmic or linear). This data set can then be used to test the inversion macros supplied with Prospa. Open the macro AnalyseT2Plot in the NNLS menu and set the parameters as shown below (note that earlier versions of the interface do not include the ‘weight bins’ option or have the ability to change the smoothing loop slope value from 0.1):

 

 

First select the smoothing parameter by pressing the “Calculate smoothing curve” button

 

 

The smoothing parameter at which the (logged) slope is 0.1 is indicated by the solid square and is at 3.577. Enter this into the Smoothing parameter text box and press the Calculate Spectrum button. Here are the results:

 

 

A reduction in the residual range can be obtained by reducing the smoothing loop slope parameter which by default is set to 0.1 (10% slope) as indicated in the publication by Fordham et al.

 

Note the increased broadening for peaks at short relaxation times relative to the original data. This reflects the weighting applied to the linearly spaced input data after it has been logarithmically binned (you can see the effect of this by unselecting the ‘Weight bins’ option).

 

 

 

The result with ‘Weight bins’ switched off – smoothing = 1.88

 

A similar result to that obtained with ‘Weight bins’ switched on is produced if log-binning is not selected, however the inversion (and smoothing calculation) takes much longer in this case.

 

As the noise level is reduced in the simulation the inversion will look closer to the original spectrum, however the log binning will always cause a preferential smoothing of the short relaxation time components. If the data can be log sampled (as it can in the T1 experiment) then better results will be obtained:
