Solvent Analysis

This tool calculates the volume fractions of solvent mixtures and similar multicomponent systems. The user selects peaks in the spectrum that are associated with each solvent. Each peak is integrated to determine the proton concentration of that peak. The proton concentration is converted to solvent fractional volume using molecular weights and densities. The regions integrated for each peak and the mathematics used to calculate the volume fractions are entered into a protocol files created by the developer. Once the developer is satisfied with the state of the procedure they can save the procedure to the Protocol Directory (a folder chosen by the user). The user can then select that protocol or ones previous created from the pull down menu.

 

Interface

Run starts the experiment. There data is first collected and then the volume fractions are calculated.

Abort stops the data acquisition at the end of the current scan.

Close saves the interface fields to files and closes the window.

Help opens this help file.

Data collection

Sample Name is the folder the data and results will be saved to.

Project Directory is the directory where the sample folders are.

Number of Scans is the number of scan the user wants to average before analysing the data.

Repetition Time is the time it takes to complete one scan. This includes data acquisition and the delay to allow for the sample to reach an equilibrium spin state.

Protocol

The Protocol Directory needs to be set to the folder where the solvent analysis protocol files are stored.

The Analysis Protocol pull down menu has a list of all the protocols saved in the Protocol directory folder. Select the protocol that matches the sample.

The results box at the bottom shows the volume fractions.

Analysis Protocol Script

The master script that is run by the interface described above calls procedures from the Analysis Protocol script. The Analysis protocol script can be modified to analyse different systems.

getCalIndex

This procedure calibrates the ppm scale to the larges peak in the NMR spectrum. This assumes that the same peak will be the largest for all concentrations measured. If the largest peak changes, then a new protocol should be created for that situation even if the solvent system is the same.  An example is shown below for ethanol and water. The largest peak is water until very high alcohol concentrations are present. Therefore the ppm scale is calibrated on the water peak and set to 4.78 ppm. Change the highlighted number in the code below to the desired calibration value.

######################################################

# 1. This procedure sets the calibrates the ppm scale

# to the larges peak.

#

#######################################################

 

procedure(getCalIndex)

# Set the largest peak to this value (in ppm)

ppm_cal = 4.78

 

endproc(ppm_cal)

 

getIntRegions

This procedure sets the integral regions in the spectrum. The integral regions are in entered into a matrix where the columns designate the range and the rows designate individual peaks. The first row will be the range of the first peak that will be integrated. The first column of that row is the lower limit of the range and the second column is the upper limit of the range. The units are in ppm.

Change the values in the highlighted region shown below. If more integrals are desired, then make sure the second value is followed by a semicolon (;). The matrix must be enclosed by brackets.

The integral regions will be relative to the calibration peak.  I recommend collecting a spectrum and referencing the ranges exactly how it is referenced in the getCalIndex procedure.

######################################################

# 2. This procedure sets the integral ranges of interest.

# Populate the matrix as shown in this example. Note that

# zero indexing is used (first peak is Peak 0).

#

# Peak 0    [start integraing,   stop integrating;

# Peak 1     start integraing,   stop integrating] 

#

#######################################################

 

procedure(getIntRegions)

 

   IntMat = [  0.25,2.00;    # Peak 0 integrate from 0.25 to 2.00 ppm

               2.7,10.0]     # Peak 1 integrate from 2.0 to 10.0 ppm

 

endproc(IntMat)

 

calcFractions

This procedure calculates the output values from the integral regions. The integrals calculated from the regions designated in the matrix above are passed to this procedure in the vector peak. The individual integrals are called using the index relative to the rows of the integration matrix IntMat. So, the integral of the first peak is peak[0], the second is peak[1], and so on. These integrals can be included in mathematical calculations using standard operations such as +,-,/,*,^,(), etc. There are additional mathematical functions that are listed in the help files. The user can create temporary variables if desired. Below is an example for determining the volume fraction of ethanol in water.

 

##########################################################

#

# 3. This procedure calculates mole, mass, and volume fractions

# based on the corrected peak integral regions

#

# The peak integrals are listed in peak

#    the integral of Peak 0 is peak[0]

#    the integral of Peak 1 is peak[1]

#

# New variables can be created to simplify equations

#

# Standard mathematical operations apply

#   + - * / () ^2

#

# The output values get packaged in "vol" as a structure (list)

#   usings the command vol->var

#   where var is a variable you want to output

#

###########################################################

 

procedure(calcFractions,peak)

 

result = struct()

 

# Calculate mole fractions

   etoh = peak[0]/3

   h2o = (peak[1]-peak[0])/2

   tot = etoh + h2o

 

   Xetoh = etoh/tot

   Xh2o = h2o/tot

 

 

# Calculate mass fraction

# Mole fraction multiplied by the molecular weith

   m_etoh = etoh*46.07

   m_h2o = h2o*18.01

   mtot = m_etoh+m_h2o

 

   mf_etoh = m_etoh/mtot

   mf_h2o = m_h2o/mtot

 

 

# Calculate volume fractions

# The volume is the mass divided by the density

   v_etoh = m_etoh/0.789

   v_h2o = m_h2o

   vtot = v_etoh+v_h2o

 

   vf_etoh = v_etoh/vtot

   vf_h2o = v_h2o/vtot

 

# Package results in structure 'vol'

   # NOTE use variables that are intuitive.

   # They will be printed in the results file

 

   result->etoh = vf_etoh

   result->h2o = vf_h2o

 

 

updateGui

This procedure writes the output into the results box at the bottom of the user interface.  Here the text is written as shown inside the quotation marks. Variables can be printed by enclosing them in dollar signs ($). The variable below $etoh,1.3f$ means print the value of the variable etoh showing three decimal places. Multiple output statements can be printed as a list. The list items are enclosed in quotation marks and separated by commas. The list is enclosed in brackets.

#######################################################

## 4. Report the values in the GUI

#  Edit the text and variables to report the correct data.

#

#  Replace $var$ with the variable you chose above and stored in "result"

#  In this example "etoh" and "h2o" were packaged in result.

# 

#  1.3f designates the number of decimal places, which is 3 here.

#

#######################################################

 

procedure(updateGui,result)

 

   assignstruct(result)

   setpar(0,8,"list",["Volume fraction ethanol = $etoh,1.3f$", "Volume fraction water = $h2o,1.3f$"])

 

endproc()

 

Example output

Volume fraction ethanol = 0.052

Volume fraction water = 0.948

Output Files

acqu.par contains all the NMR acquisition parameters used to collect the data.

data.1d contains the FID data in binary format compatible with MNova software

fid.1d contains the FID data in binary format (traditional Prospa .1d file)

fid.pt1 contains the FID plot file compatible with Prospa and SpinsolveExpert

proc.par contains the processing parameters used for the spectrum.

results.par contains the results of the analysis:

  Parameters = {

       ProjectDir = "D:\evan data\SpinsolveExpert\SolventAnalysis "

       SampleName = "1pcEtOH"

       RepTime = 1

       protDir = "D:\evan data\PulseSequences\SolventProcs"

       protocol = "EtOH-H2O"

       nrScans = 1

   }

 

   Results = {

          etoh = 0.0118768

          h2o = 0.988123

   }

results.pt1 contains the spectrum used for the analysis in the form of a plot file that can be opened in SpinsolveExpert or Prospa.

spectrum.1d contains the spectrum data in binary form.

spectrum.pt1 contains the spectrum used for the analysis in the form of a plot file that can be opened in SpinsolveExpert or Prospa.
