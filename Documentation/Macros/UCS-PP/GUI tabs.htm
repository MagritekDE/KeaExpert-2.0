User Interface Tabs

 

Predefined tabs

 

In addition to the tab defined by the pulse program you need to add a minimum of three others to make most macros function correctly. These are defined in the tab list in the pulse program files. e.g. for the 1Pulse macro

 

   tabs = ["Pulse_sequence","Acquisition",

           "Processing_Display_Std","File_Settings"]

 

Each entry (apart from the first) is the name of a macro found in the tabs folder:

 

<appdir>\Macros\Kea-PP\Tabs

 

Following are details of the variable names defined in these tabs.

 

Acquistion tab (V2&3)

 

 

 

The variables defined in this tab which can be accessed in the execpp procdure are:

 

rxGain ..........   receiver gain in dB

rxPhase .........   receiver phase in degrees

nrPnts ..........   number of collected point (in FID or in echo)

dwellTime .......   dwell time in us (called bandwidthFile in V1 of the tab interface)

nrScans .........   number of scans

flatFilter .......  whether the flat filter will be used in the digital receiver. ("yes/no")

accumulate .......  whether data will be accumulated.

usePhaseCycle ...   whether phase cycling will be applied or not.

bandwidth ........  the spectral width in kHz (read only)

acqTime ..........  total acquisition time (read only)

 

Hint: to see details about each control ctrl-click on the control to see a control information dialog.

 

Processing and Display tab (V3)

 

 

 

showTimeDomain .... show time domain ("yes/no")

filter ............ should time domain filter be applied ("yes/no")

filterType ........ type of filter to be used.

tdPhaseCorr ....... time domain phase correction ("magnitude/autophase/none")

showFreqDomain ...  show frequency domain ("yes/no")

dispRange ......... range of frequencies displayed. Also used to restrict the autophase range.

fdPhaseCorr ....... frequency domain phase correction ("magnitude/autophase/none")

 

File settings tab (V2&3)

 

 

 

dataDirectory ....  where the data is stored

expName ..........  subdirectory inside dataDirectory

expNr ............  subdirectory inside expName (can be blank)

saveData .........  whether data will be saved or not ("yes/no")

incExpNr .........  whether the experiment number will be automatically incremented after the experiment. ("yes/no")

 

Defining new tabs

 

The default pulse program compiler uses a set of tabs which have been saved to the location:

 

<appdir>\Macros\Kea-PP\Tabs

 

Each of these tabs is a modified window definition file.  The easiest way to make a new tab is to copy one of the existing ones, rename it and then modify it (usually using the window designer macro found in the GUI menu). Generally all the controls in the tab should be static-text, text-boxes, check-boxes or radiobuttons (other controls require extra work). Each control should be assigned a unique name so you can refer to it from the processing procedure.

 

An example of a tab file is shown below. This defines the acquisition tab

 

procedure(acquisition)

 

   n = window("Acquisition", -1, -1, 573, 120)

   :definecontrols(n)

   showwindow(n)

 

endproc()

 

procedure(definecontrols,n)

 

      # Define all controls with basic parameters

       statictext(1, 82, 13, "center", "Rx Gain (dB)")

       textmenu(2, 65, 31, 39, 200)

       statictext(3, 82, 62, "center", "Rx Phase (degrees)")

       textbox(4, 65, 78, 39)

       statictext(5, 190, 15, "center", "Number of")

       statictext(6, 190, 28, "center", "complex points")

       textmenu(7, 237, 17, 62, 200,

          keaCtrl:updateAcqTime();)

       statictext(8, 190, 47, "center", "Dwell time")

       statictext(9, 190, 60, "center", "(us)")

       textmenu(10, 237, 50, 62, 200,

          keaCtrl:updateFilters();)

       statictext(11, 190, 81, "center", "Number of")

       statictext(12, 190, 94, "center", "scans")

       textbox(13, 237, 82, 62)

       statictext(14, 364, 29, "center", "Flat filter")

       checkbox(15, 400, 30, "no,yes", "no",

          keaCtrl:setDwellTimeMenu();

          keaCtrl:updateFilters();)

       statictext(16, 364, 53, "center", "Accumulate")

       checkbox(17, 400, 54, "no,yes", "no")

       statictext(18, 364, 77, "center", "Phase cycle")

       checkbox(19, 400, 79, "no,yes", "yes")

       statictext(20, 492, 13, "center", "Spectral width (kHz)")

       textbox(21, 468, 27, 46)

       statictext(22, 492, 64, "center", "Acquisition time (ms)")

       textbox(23, 468, 78, 46)

 

     # Set other control parameters

       setpar(n,2,"name","rxGain",

                   "type","integer",

                   "range",[-20,70],

                   "menu",["-20","-17","-14","-11", ..."61","64","67","70"])

       setpar(n,4,"name","rxPhase")

       setpar(n,7,"name","nrPnts",

                   "type","integer",

                   "range",[2,65536],

                   "menu",["2","4","8","16","32","64", ...,"16384","32768","65536"])

       setpar(n,10,"name","dwellTime",

                   "menu",[""])

       setpar(n,13,"name","nrScans",

                   "type","integer",

                   "range",[1,1e+006])

       setpar(n,15,"name","flatFilter")

       setpar(n,17,"name","accumulate")

       setpar(n,19,"name","usePhaseCycle")

       setpar(n,21,"name","bandwidth",

                   "tag","disable")

       setpar(n,23,"name","acqTime",

                   "tag","disable")

 

endproc(23, "acquisition") # Return the number of controls and the name to appear on the tab

 

 

 

 

 

The executed acquisition tab file showing the control numbers

 

It consists of a test procedure (called acquisition in this case) which if run will display the window and allow it to be modified. This procedure calls the definecontrols procedure which defines all the controls to appear in the window. Note that the window() command itself appears in the first procedure. For this reason the definecontrols procedure must be passed the window number. It must return the number of controls defined in the window and the name you wish to appear in the tab button.

 

When generating the Kea interface, the pulse program compiler will call the definecontrols procedure in each tab file, modifying the control numbers so they are all unique and enclose the controls in a single window.

 

You can modify or generate the tab window using the window designer macro found under the GUI menu. The only requirements are:

 

  1. The control numbers must start from 1 and be sequential (1, 2, 3, ...). It is worthwhile renumbering the controls before saving them in a left to right order so that tabbing works as expected. (See designer.mac help for details).
  2. Because of the non-standard nature of this macro, when saving the modifications make sure you uncheck the option include window in procedure in the designer macro. You will still need to manually modify the window definition macro so it returns the number of controls and the name to appear on the tab.

 

 

  1. Ensure that each control has a unique content variable name (the name parameter). Since the control numbers will be automatically modified, this is the only reliable way to access the contents of the control later on.

 

Once you have completed your tab window you can add it to your pulse program macro by adding the name of the tab window file to the tabs list.
