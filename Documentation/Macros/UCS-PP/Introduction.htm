Kea pulse programming introduction

 

 

The Kea pulse programmer is a tool for making or modifying NMR pulse programs. The steps required to generate a new program are listed below

 

  1. Write the pulse program - this is a command script which when executed by the spectrometer processor generates all the necessary pulses at the correct times and then collects data when they are available. In the Kea we often only code the innermost loop of the pulse program on the spectrometer - the outer loops being executed within Prospa.

 

  1. Design a user interface. This is used for entering the various parameter required to run the experiment and for starting and stopping the experiment. Since many of the parameters are common to all experiments, the user interface description is usually only necessary for those parameters which are specific to the experiment being designed.

 

  1. Design the Prospa macro procedure, execpp, which runs the experiment and then collects and displays the data.  This procedure also controls outer loops within the experiment. The advantage of this method is that intermediate experimental results can be displayed as they are available and the spectrometer processor can be less complex than it would have to be if it controlled the complete experiment.

 

The user interface description and the pulse program definition reside in the same file – the pulse program definition macro. This is “compiled” to produce a pulse program execution macro which in addition to the code required to generate the user interface also contains the execpp procedure described in step 3.
