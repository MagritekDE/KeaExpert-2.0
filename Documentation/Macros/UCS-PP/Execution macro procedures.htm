Standard procedures in the pulse program execution macro

 

 

Once compiled, each Kea pulse program execution macro (macro_name.mac) consists of a series of procedures which have the following names and functions:

 

macro_name ..... generates and displays the GUI window and loads the last set of parameters used.

 

getseqpar .......   returns the various pulse sequence parameter lists which were originally defined in the pulse program definition macro (*_pp.mac).

 

backdoor ........ this procedure provides a way to run the pulse program without having to go via the graphical user interface. This is useful if you want to run a number of different experiments one after the other or if you want to change parameters in a single experiment without going through the bother of generating a new user interface. More details can be found in backdoor procedures.

 

windowdefinition .. this defines how the GUI window will look – normally this should not be modified – make your changes in the pulse-program description file and then recompile it.

 

execpp .......... this collects processes and saves the data. More details can be found in processing procedure.

 

other procedures .. any additional code required by the execpp procedure can be placed in separate procedures to keep the code more modular.
