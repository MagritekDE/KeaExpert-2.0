Kea experiment standard file format

 

 

 

All kea experiments should, if possible, use the standard file save procedures

 

keaFiles:save1DData(guipar, xdata, ydata, [name [, savePlot [, subDir])

 

keaFiles:save2DData(guipar, matrix, [name [, savePlot [, subDir])

 

keaFiles:save3DData(guipar, matrix, [name [, savePlot [, subDir])

 

This will result in a standard file format and allows data to be automatically loaded using the "Load Data" button in the "File settings" tab.

 

In each case the procedure requires the list of user interface parameters (guipar) and the data itself. The pair xdata,ydata for 1D and the 2D or 3D matrix for multidimensional experiments. By default data is stored in the folder

 

$workdir$\\experiment_name\\experiment_number\\subDir

 

or

 

$workdir$\\experiment_name\\subDir

 

if the experiment_number is blank. If subDir is not passed then it is not created.

 

by default, the following files are stored for each dimension if the "name" parameter is not defined. If "name" is defined, replace "data" with name. The plot files data.pt1(2) and data(2).png are stored by default, but if savePlot = 0 they will not be saved.

 

1-D

 

data.1d .... Prospa 1D data set (combining xdata & ydata)

data.pt1 ... Prospa 1D plot file

data.png ... Image of 1D plot

data.csv ... Comma delimited data set (combining xdata & ydata)

 

2D

 

data.2d ..... Prospa 2D data set (from matrix)

data2.pt2 ... Prospa 2D plot file

data2.png ... Image of 1D plot

data2.csv ... Comma delimited data set

 

3D

 

data.3d ..... Prospa 3D data set (from matrix)

data3.csv ... Comma delimited data set
