User interface control syntax

 

 

When adding controls to the Kea user interface via the pulse program definition macro there is a standard syntax which must be following when defining the interface list. This is discussed here.

 

Recall that the interface list has the following structure, repeated for each control:

 

name, label, x, y, control_type, variable_type

 

Here we will be describing the last two items for each supported control:

 

 

Button:

 

"bt",  "width, height, procedure",

 

e.g. "bt",  "20,20,:update(\"x\",1);",

 

'width' and 'height' are the button dimensions in pixels while 'procedure' can be any valid list of commands, or more commonly a call to a procedure to be executed when the button is pressed. Remember to escape any embedded quotes with backslashes.

 

Check box:

 

"cb",  "option_list"

 "cbv",  "option_list" (label placed vertically above check box)

 

e.g. "cg", "yes,no"

 

The option list is a series of options (any ascii characters) separated by commas.

 

Text box:

 

"tb",    type_and_range/type_and_width

"tbw",   type_and_range  (wide form)

"tbv",   type_and_range  (label placed vertically above text box)

"tbwv",  type_and_range  (wide form with vertically place label)

 

e.g. "tb",  "float,[-2.5,2.5]"

 

The type_and_range parameter takes the  form: “data_type, [min_value,max_value]” where data_type is either  integer , float, double or string. e.g. "integer,[ 1,1000]". (Although it makes no sense to have a range with a string – see below). There are also a set of shorthand names you can use for the type_and_range  parameter which have the following meanings:

 

pulseamp           =          "float,[-85,0]"

pulselength      =          "float,[0.5,1000]"           

freq                  =          "double,[1,400]"

reptime             =          "float,[1,1e8]"

sdelay               =          "float,[2,327670]"

ldelay               =          "float,[2,167e6]"

ldelayms           =          "float,[ 0.002,167e3]"

other or ""        =          will be ignored

 

All times are in ms apart from ldelayms parameters, which are in ms.  Setting these values correctly means you will not need to modify the final macro GUI definition each time you regenerate it.

 

“String” is a special case in that it takes only one or no number for the “range”. In this case the number is interpreted as the width of the text box in pixels. This can be used to make very long textboxes e.g. the following type and width might be useful for making a filename textbox "string",300

 

 

Text menu:

 

"tm",  "menu_list"

"tmv", "menu_list" (label placed vertically above text menu)

 

e.g.  "tm",  "[\"xy\",\"yx\",\"xz\",\"zx\",\"yz\",\"zy\"]",

 

The menu list is a series of escaped strings separated by commas and enclosed in square brackets.

 

Radio button:

 

"rb",  "spacing,option_list,label_list",

"rbv", " spacing,option_list label_list",( label placed vertically above radio button) 

 

e.g. "rbv",  "30,int,ext,other,Internal,External,Other"

 

This will create three radio buttons with the options [“int”,”ext”,”other”] and labels to the left of each button “Internal”,”External”, and “Other”. The vertical spacing between buttons will be 30 pixels.
