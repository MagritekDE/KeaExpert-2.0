pulse

 

This command generates a single or dual RF pulse with the specified amplitude, phase, duration and optionally frequency. It also includes a gate pulse which is used to control an RF amplifier.

 

Syntax – single channel option

 

pulse1(destination, amplitude (ax/tx), phase (px/tx), duration (dx/tx), [frequency (fx/tx)])

 

destination ……....  "1" for channel 1 – used for Proton and Fluorine.

"2" for channel 2 – used for X channel nuclei.

"1nb" channel 1 no blanking – in this case there is no signal to the probe, but a pulse is still produced on the transceiver output (for debugging).

"2nb" channel 2 no blanking – as for 1nb but on channel 2.

 

amplitude2 ……..... the parameter controlling the amplitude of the pulse. Parameters have the following forms:

Scalar amplitude "ax" where x is an integer greater than 0. (Units are dB, -85… 0)

Table amplitude "tx" (Integer value, range 0 ... 16383)

A floating-point constant (Units are dB, -85 … 0).

 

phase3 ................. the parameter controlling the phase of the pulse. Parameters have the following forms:

Scalar phase "px".  (Float value range 0 ... 3.99994)

Table phase "tx". (Integer value ,range 0 ... 65535 = 0-360 degrees)

A floating-point constant (Integer value range 0 ... 65535 = 0-360 degrees).

 

duration4 .............. the parameter controlling the length or duration of the pulse. Parameters have the following forms:

Scalar delay "dx". (Units are microseconds)

Table delay "tx". (Integer value, range 0 ... 65535, units are multiples of 10 ns)

A floating-point constant. (Units are microseconds).

 

frequency ...........  optional parameter controlling the frequency of the pulse. This also sets the current transmit frequency (DSP only).

If this variable is not defined, then the current transmit frequency is used. Parameters have the following forms:

Scalar frequency "fx" where x is an integer greater than 0. Units are in MHz.

Table delay "tx". (Dual integer value – see table description)

A floating-point constant (Units are in MHz).

 

Notes:

 

  1. The pulse generated by this command has a length which includes a pre-defined delay called the pulse gate overhead (pgo). This is fixed to 5 ms on the Spinsolve. This delay is used to set up the RF gate – a signal sent to the RF amplifier to switch on the DC biasing before the RF pulse is actually sent. You should account for this delay when writing the relationships list - the predefined variable “pgo” can be used.

 

  1. The pulse amplitude is a 14-bit number which provides output powers from 3 pW to 1 mW to the RF amplifier input. For scalar values this should be expressed in decibels from -85 dB to 0 dB.

 

 

  1. The pulse phase is a 16-bit number which provides phase shifts from 0 to 360 degrees. For the px and scalar option this phase range is scaled to range from 0 to 4. Normally only phase shifts of 0,1,2,3 (0, 90, 180 and 270 degrees) are entered. In the table mode the phases should be entered as 16 bit numbers i.e. phase_in_degrees/360*2^16.

 

  1. The pulse duration may vary from 0.5 ms to 1 ms (the upper limit is to prevent RF amplifier overheating). For longer low-power pulses use the txon/txoff commands.

 

  1. On FX3 systems the duration of pulse commands will be pgo + pulse_duration + 1 microsecond. The extra 1 ms at the end means that RF commands can be placed directly next to each other. On DSP systems this padding is not present and consecutive single-channel pulses should be at least 1 ms apart otherwise they will not appear correctly.  Also the pulse command lasts 300 ns after the RF pulse completes. This needs to be considered if accurate timing is required.

 

 

Syntax – dual  channel option

 

Generate a pulse on channel 1 and another on channel 2 with the same duration, but different amplitude, phases and frequencies.

 

pulse(1, amplitude1 (ax), phase1 (px), frequency1 (fx), 2, amplitude2 (ax), phase2 (px), frequency2 (fx))

 

 

amplitudeX1 …..... the parameter controlling the amplitude of the pulse. Parameters have the following forms:

Scalar amplitude "ax" where x is an integer greater than 0. (Units are dB, -85… 0)

A floating-point constant (Units are dB, -85 … 0).

 

phaseX2 .............. the parameter controlling the phase of the pulse. Parameters have the following forms:

Scalar phase "px".  (Value range 0 ... 3.99994)

A floating-point constant (Integer value range 0 ... 65535 = 0-360 degrees).

 

durationX3 .......... the parameter controlling the length or duration of the pulse. Parameters have the following forms:

Scalar delay "dx". (Units are microseconds)

A floating-point constant. (Units are microseconds).

 

frequencyX …….... the parameter controlling the frequency of the pulse. This also sets the current transmit frequency. If this variable is not defined then the current transmit frequency is used. Parameters have the following forms:

Scalar frequency "fx" where x is an integer greater than 0. Units are in MHz.

A floating-point constant (Units are in MHz).

 

Notes:

 

  1. The pulse amplitude is a 14-bit number which provides output powers from 3 pW to 1 mW. The external interface requires that this be entered on a logarithmic amplitude scale : -85 dB to 0 dB.

 

 

  1. The pulse phase is a 16 bit number which provides phase shifts from 0 to 360 degrees. For the px and constant options this phase range is scaled to range from 0 to 4. Normally only phase shifts of 0,1,2,3 (0, 90, 180 and 270 degrees) are entered.

 

  1. The pulse duration may vary from 0.5 ms to 1 ms (the upper limit is to prevent RF amplifier overheating). For longer low-power pulses use the txon/txoff commands.

 

 

  1. On FX3 systems the duration of pulse commands will be pgo + pulse_duration + 2 microseconds. On DSP systems consecutive dual channel pulses should be at least 250 ns apart. Also the pulse command lasts 600 ns after the RF pulse completes. This needs to be considered if accurate timing is required.
