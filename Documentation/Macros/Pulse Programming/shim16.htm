shim16

 

This command sets one of the Spinsolve shims to the specified value.

 

Syntax

 

shim16(address, value)

 

address .........  which gradient to send value to. Can be an integer with value 0 ... 15. The address parameter has the form “nx” where x is an integer greater than 0 or it can be one of the previously mentioned constants.

 

value ............ this parameter stores the 16 bit value to be sent to the shim controller. (-32768 to 32767). The value parameter has the form “nx” where x is an integer greater than 0. Alternatively it can be a table reference. In this case the table value pointed to by its current index will be used and value should have the form “tx” where x is an integer greater than 0.

 

Notes

 

  1. For a fixed value this command takes 2.5 ms (DSP)/ 2 ms (FX3) to complete and the gradient output will appear immediately after the end of the command (see example below). 

 

 

 

Examples

 

The following sequence shows the response time of the command (output to channel 0)

 

 ttlon(0x01)

 shim16(0,30000)

 ttloff(0x01)

 shim16(0,0)

 

 

TTL (top) and voltage across 1 ohm load resistor (bottom). The relatively slow rise time of the shim  pulse is determined by the slew rate of the voltage to current converter. Inductive loads will lead to even slower rise times.

 

Note that there is a 2 ms delay built into the command so that shim current changes appear at the end of the command. (The 2 ms delay is required for the command to propagate from the event-table to the gradient board via the serial bus). This means that shim16 commands can be adding directly after one another without additional delays. The following sequence illustrates this by generating a ramped gradient pulse using multiple shim16 commands (output to channel 0)

 

 ttlon(0x01)

 shim16(0,5000)

 shim16(0,10000)

 shim16(0,15000)

 shim16(0,20000)

 shim16(0,25000)

 shim16(0,30000)

 ttloff(0x01)

 shim16(0,0)

 

 

 

TTL (top) and voltage across 1 ohm load resistor (bottom)

 

In the following example the table option in the shim16 command is used

 

 Example pulse sequence: (PSExamples\Shim16TableTest)

 

 

############################################################

# 

# Generates a table controlled z-shim output

#

############################################################

 

procedure(pulse_program,dir,mode,pars)

 

# Expose parameters for FX3 implementation

   if(nrArgs == 3)

      assignlist(pars)

   endif

 

# Interface description (name, label, ctrl, vartype)

  interface = ["nucleus",         "Nucleus",                   "tb",  "readonly_string";

               "b1Freq1H",        "1H frequency (MHz)",        "tb",  "freq";

               "90Amplitude1H",   "Pulse amplitude (dB)",      "tb",  "pulseamp";

               "pulseLength1H",   "Pulse length (us)",         "tb",  "pulselength";

               "startShim",       "Shim initial value",        "tb",  "float,[-32768,32767]";

               "endShim",         "Shim final value",          "tb",  "float,[-32768,32767]";

               "shimSteps",       "Nr. steps",                 "tb",  "integer,[2,1000]";

               "stepDur",         "Step time (us)",            "tb",  "integer,[6,1000]";

               "repTime",         "Repetition time (ms)",      "tb",  "reptime"]

 

 

# Define the tab groups and their order

   groups = ["Pulse_sequence","Acquisition",

             "Processing_Std","Display_Std","File_Settings"]

 

# Relationships to determine remaining variable values

   relationships = ["nDataPnts    = nrPnts",

                    "a90Amp       = 90Amplitude1H",

                    "d90Dur       = pulseLength1H",

                    "tGrad        = linspace(startShim,endShim,shimSteps)",

                    "nGradSteps   = size(tGrad)",

                    "dStep        = stepDur-5.15", # Compensates for loop delays

                    "totPnts      = nrPnts",

                    "totTime      = acqTime"]

 

# These parameters will be changed between experiments

   variables = [""]

 

## Pulse sequence

 

   initpp(dir)              # Define compile directory and clear parameter list

 

   pulse(1,a90Amp,p1,d90Dur)

 

   setindex(tGrad,0)

   loop("l1",nGradSteps)

      shim16(0,tGrad)

      delay(dStep)

      incindex(tGrad,1)

   endloop("l1")

   shim16(0,0)

 

   acquire("overwrite",nDataPnts)   # Acquire FID

 

   lst = endpp(1)                # Combine commands and return parameter list

 

# Phase cycle list

   phaseList  = [0,1,2,3;   # p1 : Pulse phase

                 0,1,2,3]   # pA : Acquire phase

 

 

endproc(lst,groups,interface,relationships,variables,null,phaseList)
