delay

 

This command simply causes the pulse sequence to pause for the specified time.

 

Syntax

 

delay(duration)

 

duration ......  the parameter controlling the length or duration of the delay. Units are micro-seconds, resolution is 0.02 ms

Parameters have the following forms:

 Scalar1 delay "dx" where x is an integer greater than 0.

 Table2 delay "tx" where x is an integer greater than 0.

 A constant ranging from 0.25 to 327670.

 

Notes:

 

  1. The duration of the delay may vary from 0.25 ms to 327.670 ms. For longer delays use the wait command.
  2. In table mode one unit corresponds to 0.02 ms. However there is also an offset of 0.02 ms. So to calculate a time in ms the required table value is [t(ms)× 50]-1.

 

 

See also:

 

wait

 

 

Example

 

########################################################

#

# Demonstrates the table command by generating a series

# of TTL pulse separated by 1, 2, 3 ... 10 us.

#

########################################################

 

procedure(pulse_program,dir,mode)

 

 

# Interface description (name, label, x, y, ctrlType, varType)

   interface = ["b1Freq",       "B1 Frequency (MHz)",   "0","0",     "tbw",   "freq",

               "repTime",      "Repetition time (ms)",  "0","1",     "tbw",   "reptime",

               "nrPulses",     "Number of pulses",      "1","0",     "tb",     "integer,[1,100]"]

 

   tabs = ["Pulse sequence","Acquisition","Processing+Display","File Settings"]

 

# Relationships between pulse-sequence and GUI parameters

   relationships = ["n1 = nrPulses",

                    "t1 = (linvec(1,n1,n1)-0.26)*50-1",

                    "n2 = nrPnts",

                    "totPnts = nrPnts",

                    "totTime = acqTime"]

 

# These parameters will be changed between experiments

   variables = [""]

 

# x and y spacing between controls

   dim = [155,26]

 

# Pulse sequence

   initpp(dir) # Reset internal parameter list

                 

   setindex(t1,0)

   loop(l1,n1)

      ttlon(1) # Pulse pin 5 on the TTL port

      delay(1) # for 1 us

      ttloff(1)

      delay(t1)  # Wait a variable delay                

      incindex(t1) # Next delay

   endloop(l1)

   ttlon(1) # Last pulse

   delay(1)

   ttloff(1)

   acquire("overwrite",n2)

 

   lst = endpp() # Return parameter list

 

# Phase cycle

   phaseList = [0;0]

 

 

endproc(lst,tabs,interface,relationships,variables,dim,phaseList)

 

Note the subtraction of 0.26 ms from table values. This allows for various delays in the sequence i.e. the duration of the ttlon, ttloff, incindex, loop and endloop commands.
