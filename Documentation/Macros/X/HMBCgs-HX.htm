HMBCgs-HX

 

Description

 

Heteronuclear multiple-bond correlation (HMBC) experiment. This 2D experiment detects 1H signal along the direct dimension and X signal along the indirect dimension. It correlates chemical shifts of two types of nuclei separated from each other with two or more chemical bonds. This implementation uses gradient pulses for homospoil and coherence selection.

Parameters

 

Note that some parameters may be hidden (see View menu option Toggle parameter visibility).

 

Pulse sequence specific parameters

·         Nucleus (1H-X – read only).

·         Proton frequency – frequency of 1H-channel RF pulse (MHz).

·         Offset frequency – and offset added to the B1 frequency (Hz).

·         Pulse length –pulse duration for all pulses in microseconds (0.5 to 1000 ms)

·         90 Pulse amp – amplitude of 90 degree 1H channel pulse for the pulse duration set above in dB (-85 to 0 dB)

·         180 Pulse amp – amplitude of 180 degree 1H channel pulse for the pulse duration set above in dB (-85 to 0 dB)

·         X frequency – frequency of X channel RF pulse (MHz).

·         Offset frequency in f1 – and offset added to the X frequency (Hz).

·         90 Pulse amplitude – size of 90 degree X channel RF pulse in dB (-85 to 0 dB)

·         J(X,H) – single bond coupling constant between X and H nuclei (Hz)

·         nJ(X,H) – n bond coupling constant between X and H nuclei (Hz)

·         F1 bandwidth – acquisition bandwidth along the indirect dimension (kHz)

·         Number of steps – number of data points along the indirect dimension (integer)

·         Repetition time – time between scans (in ms)

·         Homospoil amplitude – amplitude of gradient pulses (integer, -32768 – 32767)

·         Homospoil duration – duration of gradient pulses (ms)

·         Grad ringdown time – ringdown delay after gradient pulses (ms)

·         Number of dummy cycles– dummy cycles to reach steady state (integer)

·         Pulse acqu. delay – delay at the end of rf pulses (ms)

 

Acquisition parameters

·         Receiver gain – the internal receiver gain setting in dB. Should be adjusted to avoid overload and give the best SNR. Note that the signal level will not change significantly when this parameter is changed as the displayed voltage is the probe voltage.

·         Receiver channel – specifies which nucleus should be detected (1H).

·         Receiver phase – a phase shift in degrees applied to the data before collection by the computer. Can be adjusted to give the required frequency domain phasing, overridden if autophasing is selected.

·         Number of points – the number of complex data points to be collected.

·         Dwell time – the sampling time. i.e. the time between each data point. Units are microseconds.

·         Number of scans – the number of times the experiment will be repeated and accumulated (if this option is selected).

·         Flat filter – select this if you want all frequencies to be equally weighted. Deselect if you expect all the data to be close to the central frequency.

·         Accumulate data – whether scans should be co-added to improve SNR or reduce artefacts.

·         Phase cycle – whether the phase cycle specified in the pulse sequence should be applied.

·         Bandwidth – the collected spectral frequency range centred on the B1 frequency. (kHz and read only)

·         Acquisition time – the total duration of the FID in milliseconds. Equal to the number of points multiplied by the dwell time. (ms and read only)

 

Processing parameters

·         Zero fill factor – the FID can be filled with zeroes to improve resolution in the frequency domain. The parameter determines the total length of the FID. 1 means it will not be filled with zeros, 2 it will have equal number of zeros as the FID and so on.

·         Apodization filter – a window function applied to the non-zero filled FID to improve SNR or bring out specific spectral features.

·         Filter type- the particular apodization filter to be applied.

·         Auto-phase – whether the frequency domain with be autophased or not.

 

Display parameters

·         Use ppm scale – if selected the frequency domain data will use a parts per million scale. The central frequency may be adjusted using the Set reference tool.

·         Min ppm value f1 – if the ppm scale is selected it will range from this value to Maximum ppm value along f1.

·         Max ppm value f1 – see Minimum ppm value.

·         Min ppm value f2 – if the ppm scale is selected it will range from this value to Maximum ppm value along f2.

·         Max ppm value f2 – see Minimum ppm value.

·         Display range (Hz) – if the Use ppm scale option is not selected then the display range is set by this parameter. It will center the scale about the b1 frequency. Setting this parameter to zero will display the full spectral range (i.e. the bandwidth)

 

File parameters

·         Save data – if selected the data will be saved at the end of the experiment.

·         Experiment number - if a positive integer is added here (1,2,3 …) this will be used as a subdirectory when saving the experiment.

·         Increment experiment number. If checked then the experiment number will be incremented after the experiment has been run.

 

Post processing

 

Several options are provided for post processing the collected FID and spectrum. Refer to the help in the relevant dialog for more details

·         Save – saves the currently selected plot and moves the temporary proc.par file to proc.par.

·         Export – allows plot data to be exported as either an ASCII (text) file or as a .2d Prospa file.

·         FT – Opens the FT interface for post-processing

·         Mnova – Sends the data to Mnova

·         ppm/Hz – toggles the display of parts per million or Hz in the spectrum.

·         Set ref – allows recalibration of ppm scale
