Prospa Preferences

 

 

Preferences – Menu access

 

 

 

Preferences – File location

 

All Prospa preferences are stored in the user preferences folder which has the (hidden) location:

 

C:\Documents and Settings\username\Application Data\Prospa\Preferences V3.x

 

in Windows XP, or in Vista and Windows 7:

 

C:\Users\username\AppData\Roaming\Prospa\Preferences V3.x

 

In Prospa this folder is accessible from the main file menu using the first option shown in the above figure or from the command line interface by using the variable prefdir e.g.

 

> cd prefdir

 

When Prospa is run for the first time, the user preferences folder doesn’t exist and predefined preferences stored in:

 

$appdir$\Preferences (typically C:\Program Files\Prospa\Preferences)

 

are copied to the user’s preferences folder. If you have problems starting Prospa you should first try deleting or renaming the Prospa user preferences folder in case files in this folder have been corrupted.

 

Preferences – Dialog options

 

The second option in the above figure displays the Prospa preferences dialog, which allows the user to modify some of the ways Prospa is displayed and behaves. The preference dialog allows the user to modify and save a number of parameters:

 

  1. Main menus
  2. Macro search path
  3. DLL search path
  4. Working and macro directories
  5. Plot resolution when copying to clipboard or files

Preferences – User defined menus

 

The preferences menus tab

 

Apart from the Files, Windows and Help menus all other menus in the main Prospa window are user definable.

 

(V3 in version 3 all menus can be redefined but this requires modifying the main Prospa window macro "original.mac").

 

To add a new menu, open the preferences dialog and select the Menus tab as shown above.  Press the “Add name” button to add a new name dialog which is displayed (The & sign indicates where the underscore will appear)

 

 

Press OK and the use the Up and Down buttons on the right to move the new item to the desired location in the menu bar (top corresponds to the left side on the menu bar and bottom to the right side).

 

 

Next press the “Add folder” button and move into the macro directory you want to add. Normally this will be relative to the application directory appdir (this is where Prospa is stored) but it could be somewhere else, as in the following case (the absolute option is important if you are specifying a folder on another drive).

 

 

Once selected, use the up and down buttons to move the folder so it is on the same row as the menu name you added:

 

 

Finally press the Save button to update the main menu.

 

 

Any macros you have, or will save, under the new menu, will now be accessible.

 

Drag and drop option

 

An alternative method for adding new menu items is to drag a drop a folder icon onto the Prospa title bar. A dialog will pop-up asking for the menu name to be associated with this folder and the new menu will then be added. If the folder is inside the Prospa install directory a relative path will be used, while outside this folder an absolute path is taken. This is new method is much faster than the original one if the folder you want to add is visible on the desktop.

 

Resetting the menus to their default configuration

 

If you press the reset button in the Menus tab then the menu will revert to the default configuration and all added menus will be removed. This is useful if you have more than one Prospa version installed and you want to switch between the two.

 

 

Saving and Loading menus

 

If you have a menu you would like to save press the Save button and choose a name for the menu. The menu will be stored in the following folder:

 

$prefdir$\Menus

 

Similarly if you have saved a menu you can load it in the preferences window by pressing the load button. This automatically updates the Prospa main menu as well.

 

 

 

 

Preferences – macro search path

 

 

When you access a macro command from the CLI or from another macro you do so by invoking the macro’s name with or without a procedure:

 

testmacro()

 

or

 

testmacro:aProcedure()

 

 

The macro directory is not included in the command. To ensure that the macro can be found, Prospa uses the following search path:

 

  1. The current working directory (as set using the cd() command)
  2. The folder in which the macro containing the command is found.
  3. Any subfolders in folder 2.
  4. The search paths indicated in the preferences macros tab (as shown above).

 

(Note the order change in V3) 

 

In the above example the user-defined search path includes all files and folders inside the Macros folder found under the Prospa application directory i.e. $appdir$\Macros.  This allows macros calls from within Prospa and the supplied menus to access commands stored in the coreMacros folder. (coreMacros is found in $appdir$\Macros\coreMacros)

 

Additional search paths can be appended by pressing the “Add” button and selecting a directory to add using the method described in the user-defined menus preferences section. These might be useful if you add your own macros in a folder elsewhere on your computer.

 

Preferences – DLL  search path

 

 

When a DLL command is invoked – for example t2fit(), Prospa will search the path in the order specified in this tab. By default, only one folder is searched - $appdir$\dlls\, however if you write your own DLLs you may wish to place them elsewhere so they don’t get erased accidentally when you upgrade to a new version of Prospa.

 

(V3 The function within the tab option “Plot”, which was available in V2.2, may now be found in the 1D and 2D Pref menu) 

 

Preferences – Favourites

 

 

At present this tab controls only two folders – the experiment working directory and the user macro directory. Note that the former is independent of the current directory. These directories are accessible from all file and folder browser dialogs. From within a macro or CLI the working directory may be accessed using the variable "workdir" and the macro directory with "macrodir". If the macrodir includes the variable appdir you will need to use the following command to make use of macrodir

 

cd(evalsubexp(macrodir))

 

Preferences – Copy Plot

 

 

When plots a copied a bitmap is placed on the clipboard. By default this will have the same size as the original, however if you require more resolution increase the scale factor in this text box. e.g. 2 will double the resolution in both dimensions. However it will also require four times the memory so take care when copying larger images or plots.

 

Scale factor = 1, image size 100%

Scale factor = 2, image size 50%

 

Note that this factor is also applied when saving plot or images to picture files.
