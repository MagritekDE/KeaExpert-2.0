1      Introduction

 

Complex software without good documentation can be difficult or impossible to use. In Prospa we have supplied a large number of HTML files to aid in understanding and using this software package. These files are stored in the <prospa>\Documentation folder where <prospa> is the installation directory for the package. Documents are organised in a simple folder hierarchy: topics->subtopics->help_files. The contents of any particular path through this hierarchy are displayed in the Help Viewer which appears when you select Help viewer from the Prospa Help menu.

 

 

Topics, subtopics and help files are displayed in three list boxes on the lower left. You can select any entry in these list boxes to display a particular help file from the documentation folder. In this example we are displaying the file openserial stored in the location:

 

<prospa>\Documentation\DLLs\Serial\openserial.html

 

 

2      Help file categories

 

 

 

The supplied help files topics fall into several categories:

 

 

1.      General user information. This gives an overview of the software and details about new features in the latest version. File formats and frequently asked questions are also included here.

2.      User interface help. This describes the operation of the default Prospa interface (original mode). 

3.      Command help  Prospa contains more than 400 built-in macro (scripting) commands  each of these has a help file entry.

4.      Class commands. The Prospa interface is built up of windows and different types of window controls. These controls can be interrogated or modified using a class based command syntax.

5.      DLL help  Prospa supports plugin DLLs which provide additional commands for specialist purposes.

6.      Macro help. Prospa has a large number of macros or scripts to support its operation. These are documented in the Macros folder.

7.      Details on how to write macros and design user interfaces.

 

Other topics may be added in the future

 

The built-in macro commands are organised rather differently from the other files. On disk all command files appear in the same topic folder (<prospa>\Documentation\Commands) However in the macro they appear organised in a number of subtopics. This is because some commands appear in more than one subtopic.

 

Unlike other help systems the Prospa documentation does not require special compilation. New help files can be added to the appropriate topic folder and will appear automatically in the help viewer. The only modification which may be required before they become visible is to check if there is a file called helplist.lst in the folder. This overrides the simple display of everything in the folder and allows the order to be modified and dividers to be placed between the entries (using the SPACE keyword). 3      The HTML viewer

 

To the right of the window is the HTML viewer. This is a standard ActiveX browser control. In addition to being able to display disk based HTML files it can also link to web sites if there is an appropriate link in the HTML file.

 

It also includes a standard popup menu which supports a range of functions including copying text, jumping forward and back through links and previewing or printing the current page.

 

 

A menu option (Edit->Run Selection) also exists allowing selected macro code to be run from the browser. This is useful when working through the examples found in the Commands help files.

 

Selecting the text in the box3d help file as shown, and then executing this selection (Ctrl+R) will generate the plot on the right  however it will be displayed in the current 3D plot window.

 

4      Searching the help files

 

The HTML help files can be searched for specific strings using the help controls found at the top left of the interface.

 

 

Enter the string to search for in the Search string textbox (in this case we are searching for viewdistance). Choose if you want the search to be case sensitive or not and then specify the search range using the corresponding text-menu control. The range defaults to the currently selected HTML file but it can also be by subtopic, topic or alltopics the last option searches everything. When the search has completed the number of files searched and the number of matches found will be reported in the status bar at the bottom of the window

 

 

The found files will be displayed in the Help files list box which will now be relabelled Search matches:

 

 

These matches are sorted by subtopic folder and within this by the number of matches found. If you want the complete list to be sorted by matches found then select the Edit menu option Sort by number of matches. Alternatively you can sort by path\fileName by choosing the Sort by name option in the menu.

 

If you click on one of these search entries the corresponding help file will be displayed with the first text match visible and highlighted in orange. Other matches will be highlighted yellow.

 

 

If there are multiple matches in one document (there are two here) then you can step between them using the up and down arrow keys.

 

 

The current match is always coloured orange.

 

To revert to the normal interface view, without the highlighting or search matches displayed, press the Exit search button.
