Carbon

 

Description

 

The 1D Carbon protocol records 13C spectra using a single 45 degree excitation pulse. 

 

Pulse sequence

 

The excitation pulse is preceded by a pulse train on the proton channel for signal enhancement due to the Nuclear Overhauser Effect (NOE). During acquisition composite pulse decoupling (CPD) is performed on the proton channel (the WALTZ-16 pulse train is used for both NOE and CPD).

 

Parameters

 

Note that some parameters may be hidden (see View menu option Show all parameter entries).

 

Pulse sequence specific parameters

         Nucleus (13C read-only).

         Carbon frequency frequency of channel 2 (X) RF pulse (MHz)

         Proton frequency frequency of channel 1 RF pulse (MHz)

         Offset frequency an offset added to the Carbon frequency (Hz).

         Repetition time time between scans (in ms)

         13C pulse amplitude size of channel 2 45 degree RF pulse (in dB: -85 to 0 dB)

         13C pulse length 45 degree pulse duration on channel 2 (in ms)

         Decouple amplitude amplitude of the decoupling pulse during data acquisition (in dB: -85 to 0)

        1H 180 pulse length length of 180 pulses used in the NOE and CPD pulse trains (in ms)

       NOE delay time duration of the NOE pulse train in milliseconds.

       NOE power amplitude of the pulses in the NOE pulse train (in dB: -85 to 0)

         Pre-acquisition delay delay between end of 45 degree 13C RF pulse and start of acquisition (in microseconds). This time is fixed to minimise p1 errors.

 

Acquisition parameters

         Receiver gain the internal receiver gain setting in dB. Should be adjusted to avoid overload and give the best SNR. Note that the signal level will not change significantly when this parameter is changed as the displayed voltage is the probe voltage.

         Receiver channel specifies which nucleus should be detected (13C in this case)

         Receiver phase a phase shift in degrees applied to the data before collection by the computer. Can be adjusted to give the required frequency domain phasing if autophasing is disabled.

         Number of points the number of complex data points to be collected.

         Dwell time the sampling time. i.e. the time between each data point. Units are microseconds.

         Number of scans the number of times the experiment will be repeated and accumulated (if this option is selected).

         Flat filter select this if you want all frequencies to be equally weighted. This is the default setting.

         Accumulate data whether scans should be co-added to improve SNR or reduce artefacts.

         Phase cycle whether the phase cycle specified in the pulse sequence should be applied.

         Bandwidth the collected spectral frequency range centred on the B1 frequency. (kHz and read only)

         Acquisition time the total duration of the FID in milliseconds. Equal to the number of points multiplied by the dwell time. (ms and read only)

 

Processing parameters

         Zero fill factor the FID can be filled with zeroes to improve resolution in the frequency domain. The parameter determines the total length of the FID. 1 means it will not be filled with zeros, 2 it will have equal number of zeros as the FID and so on.

         Apodization filter a window function applied to the non-zero filled FID to improve SNR or bring out specific spectral features.

         Filter type- the particular apodization filter to be applied.

         Auto-phase whether the frequency domain with be autophased or not.

 

Display parameters

         Use ppm scale if selected the frequency domain data will use a parts per million scale. The central frequency may be adjusted using the Set reference tool.

         Minimum ppm value if the ppm scale is selected it will range from this value to Maximum ppm value.

         Maximum ppm value see Minimum ppm value.

         Display range (kHz) if the Use ppm scale option is not selected then the display range is set by this parameter. It will center the scale about the 13C B1 frequency. Setting this parameter to zero will display the full spectral range (i.e. the bandwidth)

 

File parameters

         Save data if selected the data will be saved at the end of the experiment.

         Experiment number - if a positive integer is added here (1,2,3 ) this will be used as a subdirectory when saving the experiment. (No longer used)

         Increment experiment number. If checked then the experiment number will be incremented after the experiment has been run. (No longer used)

 

 

Post processing

 

Several options are provided for post processing the collected FID and spectrum. Refer to the help in the relevant dialog for more details

 

Time domain only

        SNR measures the signal to noise ratio of the time domain data. Based on measuring the maximum amplitude and dividing by the standard deviation of the data at the end of the FID.

         FT - opens a dialog allowing the FID data to be apodized, phased, deconvolved, offset, and transformed.

        MNova -  exports the FID to MNova for processing and display.

 

Frequency domain only

         Phase allows the adjustment of zeroth and first order phase correction parameters for the spectrum.

         Apodize allows a variety of apodization filters to be applied to the spectral data.

         SNR measures the signal to noise ratio of the spectrum. Based on measuring the maximum amplitude and dividing by the standard deviation of the data at the end of the FID or the start of the spectrum.

         ppm/Hz toggles the display of parts per million or Hz in the spectrum.

         Set ref. allows the zero point in the spectrum to be set if in ppm mode.

         Picking allows a list of peaks to be selected automatically or manually in the spectrum.

         Integ peaks which have been previously selected can be integrated to determine the relative contribution from each spin.

 

Both domains

         R/C toggles the display of real or complex data.

         Save saves the currently selected plot and moves the temporary proc.par file to proc.par.

         Export allows plot data to be exported as either an ASCII (text) file or as a .1d Prospa file.
