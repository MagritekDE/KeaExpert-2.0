Layout

 

These macros control how the Prospa application looks i.e. which windows are displayed and where. Normally the “original” layouts will be used as these are the most versatile. You can always reset the display without losing any work by choosing one of the originalxD options.

 

Detail for all the options are shown below.

 

allinone ........ removes the current interface and replaces it with an “all in one” interface i.e. all the plots CLI etc are inside the same window.

cliWin .......... generates and displays a separate command line interface window.

editWin .......   generates and displays a separate text editor window.

original1D ...   reorganises the standard windows layout to just display the 1D plot and hide the 2D and 3D plots.

original2D ...   reorganises the standard windows layout to just display the 1D and 2D plots and hide the 3D plot.

original3D ...   reorganises the standard windows layout to display all plots; 1D, 2D and 3D.

original ........   this regenerates the complete “original” (i.e. Prospa 2.x) interface. All work will be lost.

plot1dWin ...   generates and displays a separate 1D plot window.

plot2dWin ...   generates and displays a separate 2D plot window.

plot3dWin ...   generates and displays a separate 3D plot window.

simple ..........   removes the current interface and replaces it with a simple menu from which different windows can be generated.

 

If you modify the “original” layout manually and exit the program, then the next time you run Prospa it will remember the last layout you chose and display that. If you wish you can save this layout to a file and access it via the menu interface. Just select the menu option "File-›Save window layout", It can be reloaded at a later time using “File-›Load window layout”.
