makeVelocityMap

 

This macro generates a velocity map by analysing a set of q-space images obtained using a PGSE + imaging  sequence in which the PGSE gradient has been incremented between each image. Both 2 dimensional and 3 dimensional data can be processed with this macro.

 

 

Before this macro can be run, the complex 2D or 3D input data set must have been either loaded from a file or generated using the nDFourierTransform macro.

 

Once loaded:

 

  1. First select the q-space data-set from the pop-up menu to the right of the “q-space matrix” field. Next select a name for the 1D or 2D velocity and image matrix maps and the q dimension (one of x, y or z).

 

  1. Enter a noise threshold from the first line/plane of the q-space data set. Velocities will only be calculated for points higher than the noise threshold. You will probably have to calculate a magnitude image to do this, since the q-space data must be complex. (To do this, display the first line/plane of the q-space data set and then run the  magnitude1d or magnitude2d macro from 1D or 2D menu).

 

  1. Choose a zero-fill size for each q-space vector from the “Zero fill” popup menu (it must be a power of 2).

 

  1. Enter the maximum gradient value, and large and small delta value in SI units into the remaining fields.

 

  1. Press “Calculate” to generate the velocity plot/image.

 

  1. For 3D q space data you can press the 1D view button to display the raw data used to determine the velocity for a particular point in the image.

 

 

Note that all data points in the velocity image have units of mm/s.
