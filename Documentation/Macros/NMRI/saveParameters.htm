saveParameters

 

Loading data into Prospa using the importData macro creates a global matrix (e.g. flow3D) containing the imported data and a global parameter list with the same name but suffix .pl (e.g. flow3D.pl). The parameter list will contain all the information you have entered about the data set including the directory where it is stored.

 

You can save this parameter list back onto disk using the saveParameter macro also found in the NMRI menu.

 

 

 

This window should display the current data set (this is identified by the global variable curmatrix), along with its dimension (3D in this case) and dataset name. The data set name is included to ensure that you save the correct data set. If you are not displaying the correct dataset then choose the correct dimension or matrix name until you are. Finally press the Save button which as a final check will show the contents of the current data set folder:

 

 

Press the “Select folder” button to save the new “exp.par” to this folder. Once saved you will now be able to load your data file without typing in the parameters again, since they will be read automatically from the exp.par file you have just saved.

 

Using the other NMRI macros.

 

The philosophy behind the parameter macro idea is as follows: whenever a data set is imported it takes with it a parameter list (e.g. flow3D.pl) that can be passed to the next processing step. That processing step will result in the addition of new parameters that will be added to the input parameter list to generate a new list, which can then be passed with the processed matrix to any subsequent processing macro. e.g.  if the output matrix of a Fourier transform step is flow3DFt, then the new parameter file will be called flow3DFt.pl and will contain, in addition to the previously mentioned parameters, those shown below.

 

 

   matrixFT = "FlowFt"

   1dft = "FTEcho"

   2dft = "FTEcho"

   3dft = "none"

   1dzf = 256

   2dzf = 256

   3dzf = 8

   1dfltr = "sinebellsquared"

   2dfltr = "sinebellsquared"

   3dfltr = "none"

   magnitude = "no"
